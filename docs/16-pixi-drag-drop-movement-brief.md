# 16. Pixi 드래그 이동 보정 브리핑

## 1. 목적

모바일/터치 환경에서 유닛을 이동할 때 손을 뗀 위치가 정확히 칸 안에 들어가지 않아도 이동이 실패하지 않도록 개선했습니다.

## 2. 적용 파일

```text
apps/web/src/game-client/pixi/pixiBoardRuntime.ts
apps/web/src/game-client/pixi/pixiDragRuntime.ts
```

## 3. 변경 내용

### 3.1 가장 가까운 보드 칸으로 드롭 보정

기존에는 `getCellIndexAtPoint()`가 손을 뗀 좌표가 셀 내부에 있을 때만 대상 칸을 반환했습니다.

이제 드래그 종료 시 `getNearestCellIndexForDrop()`을 사용합니다.

동작 방식:

- 손을 뗀 위치가 정확히 셀 안이면 해당 셀 사용
- 셀 사이 간격이나 테두리 근처에 놓으면 가장 가까운 셀 중심 기준으로 보정
- 보드에서 너무 멀리 벗어나면 기존처럼 이동 취소

### 3.2 이동 실패 감소

모바일에서 손가락이 필드 경계에 걸쳐 있거나 셀 사이 간격에 떨어져도 가까운 칸으로 이동합니다.

### 3.3 이동 애니메이션 복구

이동/중첩/자리교환 성공 시 기존의 walk ghost 애니메이션을 유지합니다.

애니메이션은 실제 상태 반영 전에 먼저 재생되고, 애니메이션 종료 후 `refs.state`를 갱신합니다.

## 4. 확인 필요

```bash
pnpm typecheck
pnpm build:web
```

브라우저 확인 항목:

- 유닛을 셀 중앙에 드롭하면 정상 이동되는지
- 유닛을 셀 사이 간격에 드롭해도 가까운 칸으로 이동되는지
- 보드 테두리 근처에서 손을 떼도 가까운 칸으로 보정되는지
- 보드에서 멀리 밖으로 던지면 이동 취소되는지
- 이동/중첩/자리교환 애니메이션이 보이는지
- 신화 중첩 불가, 칸 가득 참 메시지는 기존처럼 유지되는지
