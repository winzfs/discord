# 23. 신화 조합 메뉴 모바일 레이아웃 정리

## 1. 목적

모바일 화면에서 신화 조합 메뉴가 아래로 드래그되지 않고, 재료 라벨이 너무 산만하게 보이는 문제를 정리했습니다.

추가로 메뉴를 열 때와 드래그할 때 렉이 걸리는 문제를 줄이기 위해 조합 목록을 가상 스크롤 + 지연 렌더 방식으로 변경했습니다.

## 2. 적용 파일

```text
apps/web/src/game-client/pixi/pixiMythicMenuView.ts
apps/web/src/game-client/pixi/pixiVirtualScrollScheduler.ts
apps/web/src/game-client/pixi/gameTheme.ts
```

## 3. 스크롤 수정

기존 구조에서는 각 조합 행이 `pointertap`을 직접 받아 스크롤 드래그와 충돌할 수 있었습니다.

변경 후:

- 스크롤 영역에 투명 hit area 추가
- 행의 `pointertap` 제거
- 조합 가능 행은 `pointerdown → pointermove 거리 체크 → pointerup` 방식으로 처리
- 드래그 거리가 일정 이상이면 조합 터치로 보지 않음
- 모바일에서는 목록 영역을 위아래로 드래그해 스크롤

## 4. 렉 개선

기존에는 신화 메뉴를 열 때 모든 조합 행을 한 번에 생성했습니다.

문제점:

```text
전체 조합 행 생성
각 행 패널 생성
각 행 재료 텍스트 생성
각 행 등급 라벨 그래픽 생성
드래그 중에도 행 제거/재생성 반복
```

변경 후:

```text
현재 보이는 행 + 위아래 여유 행만 생성
드래그 중에는 목록 위치만 이동
드래그 종료 후 보이는 범위만 다시 생성
휠 스크롤은 requestAnimationFrame으로 갱신 묶기
재료 진행도는 recipeId 기준으로 캐시
```

핵심 구현:

- `createVirtualRowRenderer()` 추가
- `VISIBLE_ROW_BUFFER = 2` 적용
- `pixiVirtualScrollScheduler.ts` 추가
- `pointermove` 중 `renderVisibleRows()` 직접 호출 제거
- `pointerup`, `pointerupoutside`, `wheel`에서만 렌더 갱신 요청
- `progressCache`로 `getMythicIngredientProgress()` 반복 호출 감소
- `spacer` 그래픽으로 전체 스크롤 높이 유지

## 5. 레이아웃 정리

현재 표시 방식:

```text
재료 2/4 · 부족 2
✓ [일반] 헬릭스 경비병      · [영웅] 헬릭스 전술 방패병
· [영웅] 정커 스크랩 정비공  · [전설] 옴닉 요새 집행기
```

핵심 원칙:

- 한 줄에 재료 2개씩 표시
- 총 2줄로 최대 4개 재료 표시
- 등급 라벨은 유닛 이름 바로 앞에 별도 색상 블록으로 표시
- 라벨 내부 글자는 전부 흰색 계열로 표시하고 어두운 그림자를 추가
- 글씨 크기는 너무 작지 않게 11 기준 유지
- 우측 등급 점 표시는 제거
- 조합 가능 행만 배경을 살짝 강조

## 6. 등급 색상 기준

공통 등급 색상 함수 `gameTheme.gradeColor()` 기준으로 통일했습니다.

```text
희귀: 밝지만 너무 하얗지 않은 하늘색 0x9eefff
영웅: 진하지만 너무 검지 않은 보라색 0x4b1aa8
전설: 노란색
신화: 주황색
일반: 흰색
```

기존에는 보드 유닛 색상이 `gameTheme.gradeColor()`를 쓰고, 신화 메뉴는 자체 색상 함수를 써서 서로 다르게 보였습니다.

이제 신화 메뉴도 공통 등급 색상 함수를 사용합니다.

## 7. 테스트 체크리스트

- 신화 메뉴를 눌렀을 때 멈춤이 줄었는지 확인
- 신화 메뉴 목록을 아래로 드래그할 수 있는지 확인
- 드래그 중 프레임 드랍이 줄었는지 확인
- 드래그가 끝난 후 행이 정상적으로 교체되는지 확인
- 휠 스크롤 시 행이 정상적으로 교체되는지 확인
- 조합 가능 행을 짧게 터치하면 조합되는지 확인
- 드래그 중 실수로 조합되지 않는지 확인
- 재료 라벨이 한 줄에 2개씩 보이는지 확인
- 등급 라벨이 유닛 이름 바로 앞에 색상 블록으로 보이는지 확인
- 라벨 내부 글자가 어두운 배경/밝은 배경 모두에서 읽히는지 확인
- 보드 위 희귀 유닛은 밝은 하늘색, 영웅 유닛은 진한 보라색으로 구분되는지 확인
- 신화 조합 메뉴 라벨도 보드 유닛 색상과 같은 기준으로 보이는지 확인
- 글씨가 너무 작게 보이지 않는지 확인
- 하단 안내 문구가 목록을 가리지 않는지 확인
