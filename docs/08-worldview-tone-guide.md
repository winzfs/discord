# 08. 세계관 및 톤 가이드

## 1. 최종 방향

이 프로젝트는 **운빨존많겜 스타일의 랜덤 디펜스**를 기반으로 하되, 메인 플레이는 **싱글플레이 위주**로 설계합니다.

2인 협동은 가능하면 좋은 확장 요소로 남기고, 초기 MVP의 필수 조건으로 두지 않습니다.

핵심 방향:

> 싱글플레이 중심의 운빨 랜덤 디펜스에 오버워치 영웅, 역할군, 스킬, 궁극기 감성을 입힌 비상업 팬게임.

## 2. 프로젝트 성격

- 비상업 팬게임
- 디스코드 커뮤니티 내부 놀이용 웹게임
- 수익화 없음
- 광고 없음
- 유료 재화 없음
- 유료 시즌패스 없음
- 공식 서비스 아님
- GitHub + Cloudflare 기반으로 운영

권장 고지 문구:

```text
이 프로젝트는 오버워치 팬 커뮤니티를 위한 비상업 팬게임입니다.
오버워치 및 관련 명칭, 캐릭터, 설정은 Blizzard Entertainment의 자산입니다.
본 프로젝트는 Blizzard Entertainment와 공식적으로 관련이 없으며, 후원·제휴·승인을 받은 서비스가 아닙니다.
```

짧은 버전:

```text
비상업 팬게임입니다. Blizzard Entertainment와 공식 관련이 없습니다.
```

## 3. 장르 기준

기본 게임 구조는 운빨존많겜류 랜덤 디펜스의 핵심 재미를 따릅니다.

주요 구조:

1. 싱글플레이 기본 진행
2. 제한된 필드 슬롯
3. 랜덤으로 영웅 소환
4. 같은 등급/조건의 영웅 합성
5. 상위 등급 영웅 획득
6. 웨이브 방어
7. 보스 웨이브 돌파
8. 최대한 오래 생존하거나 목표 웨이브 클리어
9. 결과 점수와 웨이브를 랭킹에 반영

## 4. MVP 기본 모드

MVP에서는 여러 모드를 만들지 않습니다.

기본 모드 하나만 만듭니다.

```text
싱글 랜덤 웨이브 디펜스
```

이 모드에서 다음을 모두 경험할 수 있어야 합니다.

- 영웅 소환
- 영웅 합성
- 웨이브 방어
- 보스 처치
- 점수 저장
- 랭킹 반영

2인 협동은 MVP 이후 확장 후보입니다.

## 5. 2인 협동의 위치

2인 협동은 메인이 아니라 확장 콘텐츠로 둡니다.

추후 2인 협동을 넣는다면 다음 방식이 좋습니다.

- 친구 초대 링크로 방 생성
- 각자 자기 필드 보유
- 팀 생명 또는 공통 보스 체력 공유
- 같은 웨이브를 함께 방어
- 결과는 듀오 랭킹에 따로 저장

하지만 초반 개발에서는 실시간 동기화가 부담이므로 보류합니다.

우선순위:

```text
1순위: 싱글 플레이 완성
2순위: 싱글 랭킹/시즌
3순위: 2인 협동 확장
```

## 6. 게임 한 판 흐름

1. 유저가 디스코드 로그인
2. 게임 시작 버튼 클릭
3. 싱글 플레이 필드 생성
4. 기본 자원 지급
5. 랜덤 영웅 소환
6. 웨이브 시작
7. 영웅이 자동 전투
8. 적 처치 보상으로 자원 획득
9. 영웅 추가 소환 또는 강화
10. 같은 등급 영웅 합성
11. 보스 웨이브 돌파
12. 생명 또는 서버 코어 내구도가 0이 되면 종료
13. 최종 웨이브/점수 저장
14. 개인 최고 기록 및 서버 랭킹 반영

## 7. 오버워치 영웅 적용 방향

사용자는 오버워치 영웅과 스킬을 팬게임 요소로 사용하는 방향을 원합니다.

따라서 기획상 영웅은 오버워치 영웅을 기반으로 구성할 수 있습니다.

다만 개발 구조는 나중에 이름을 바꾸거나 별도 명칭으로 전환하기 쉽도록 `id`, `displayName`, `role`, `skills`, `tags`를 분리합니다.

예시 구조:

```ts
type HeroDefinition = {
  id: string;
  displayName: string;
  role: 'tank' | 'damage' | 'support';
  grade: 'normal' | 'rare' | 'epic' | 'legendary' | 'mythic';
  attackType: 'single' | 'area' | 'support' | 'control';
  skillIds: string[];
  tags: string[];
};
```

중요:

- 코드 로직에서 영웅 이름으로 분기하지 않습니다.
- 영웅별 효과는 `skillId`와 `tags`로 처리합니다.
- 밸런스 값은 데이터 파일로 분리합니다.
- 이미지/아이콘은 가능하면 직접 제작한 팬아트 또는 별도 스타일의 아이콘을 사용합니다.

## 8. 역할군 변환

오버워치의 역할군을 랜덤 디펜스에 맞게 변환합니다.

| 역할 | 디펜스 내 기능 |
|---|---|
| 탱커 | 적 지연, 방어선 보호, 실드, 도발, 보스 버티기 |
| 딜러 | 단일 피해, 범위 피해, 연속 공격, 보스 딜링 |
| 지원 | 공격속도 증가, 피해 증가, 회복, 자원 보조, 부활성 효과 |

MVP에서는 세 역할만 사용합니다.

- 탱커
- 딜러
- 지원

제어형, 행운형 같은 별도 역할은 만들지 않습니다. 필요하면 영웅 태그나 스킬 효과로 처리합니다.

## 9. 영웅 등급

운빨존많겜식 성장 체감을 위해 영웅 등급을 둡니다.

추천 등급:

| 등급 | 설명 |
|---|---|
| 일반 | 자주 등장, 합성 재료 |
| 희귀 | 초반 핵심 전력 |
| 영웅 | 조합 중심 유닛 |
| 전설 | 판을 바꾸는 핵심 영웅 |
| 신화 | 낮은 확률의 강력한 영웅, 추후 확장 |

MVP에서는 4단계만 추천합니다.

```text
일반 / 희귀 / 영웅 / 전설
```

신화는 추후 확장용으로 남겨둡니다.

## 10. 영웅 소환

기본 구조:

- 자원을 소모해 랜덤 영웅을 소환합니다.
- 빈 슬롯이 있어야 소환할 수 있습니다.
- 소환 비용은 횟수에 따라 증가할 수 있습니다.
- 영웅 등장 확률은 등급별 확률표를 따릅니다.

예시 확률:

| 등급 | 확률 |
|---|---:|
| 일반 | 65% |
| 희귀 | 25% |
| 영웅 | 8% |
| 전설 | 2% |

실제 수치는 `balance.ts` 또는 DB 설정으로 분리합니다.

## 11. 합성 규칙

운빨존많겜식 단순 합성을 기준으로 합니다.

MVP 추천 규칙:

```text
같은 등급 영웅 3명 합성 → 한 단계 높은 등급의 랜덤 영웅 1명
```

예시:

```text
일반 3명 → 희귀 1명
희귀 3명 → 영웅 1명
영웅 3명 → 전설 1명
```

합성 결과는 완전 랜덤으로 시작합니다.

나중에 필요하면 같은 역할 확률 보정, 특정 영웅 조합 보너스, 시너지 보너스를 추가합니다.

## 12. 스킬 적용 방향

각 영웅은 기본 공격과 스킬을 가질 수 있습니다.

스킬은 크게 네 종류로 나눕니다.

| 스킬 타입 | 설명 |
|---|---|
| 공격 스킬 | 단일/범위/연속 피해 |
| 제어 스킬 | 둔화, 기절, 밀침, 방어력 감소 |
| 지원 스킬 | 공격속도 증가, 피해 증가, 회복, 자원 보조 |
| 궁극기형 스킬 | 일정 조건에서 강한 효과 발동 |

MVP에서는 모든 영웅에게 복잡한 스킬을 넣지 않습니다.

추천:

- 일반/희귀: 기본 공격 위주
- 영웅 등급: 간단한 패시브 또는 확률 효과
- 전설 등급: 확실한 고유 스킬 또는 궁극기형 효과

## 13. 궁극기 감성

오버워치 팬게임 느낌을 살리려면 궁극기 감성이 중요합니다.

하지만 조작형 궁극기보다는 디펜스 장르에 맞게 자동/조건부 발동을 추천합니다.

발동 조건 후보:

- 일정 공격 횟수마다 발동
- 일정 시간마다 발동
- 보스 등장 시 발동
- 적이 많이 쌓였을 때 발동
- 합성 직후 발동

예시:

```text
전설 영웅이 10초마다 광역 스킬 사용
지원 영웅이 보스 등장 시 전체 공격속도 증가
탱커 영웅이 서버 코어 피해를 1회 막음
```

## 14. 필드 구조

MVP 추천 필드:

```text
4 x 4 슬롯형 필드
```

이유:

- 모바일에서도 보기 좋음
- 합성 관리가 쉬움
- 운빨존많겜식 슬롯 관리와 잘 맞음
- 너무 넓지 않아 초반 구현이 쉬움

대안:

```text
3 x 5 슬롯형 필드
```

## 15. 적 구성

적은 오버워치 캐릭터보다 게임 전용 적으로 구성하는 것이 좋습니다.

추천 적:

| 적 이름 후보 | 특징 |
|---|---|
| 잡버그 | 기본 적 |
| 렉덩어리 | 느리지만 튼튼함 |
| 핑폭탄 | 빠르게 지나감 |
| 트롤봇 | 랜덤 방해 효과 |
| 패치오류체 | 죽으면 작은 버그로 분열 |
| 서버크래셔 | 기본 보스 |
| 새벽큐 악몽 | 이벤트 보스 |

## 16. 결과 화면 톤

결과 화면은 디스코드에 공유하고 싶을 정도로 유쾌하게 만듭니다.

예시:

- 오늘의 팀합: 이상하게 좋음
- 조합 상태: 딜러만 나왔는데 왜 이김?
- 힐러가 안 나왔지만 멘탈로 버팀
- 탱커 없이 20웨이브 돌파. 이게 맞나?
- 전설 하나가 서버를 살렸다
- 억까를 이긴 자
- 이 판은 하이라이트 감이다
- 서버 코어가 간신히 살아남았다
- 랜덤이 억까했지만 손가락이 캐리했다

## 17. 대시보드 명칭 후보

| 일반 명칭 | 유쾌한 명칭 후보 |
|---|---|
| 랭킹 | 오늘의 캐리榜, 서버 하이라이트 |
| 내 기록 | 내 하이라이트 기록 |
| 게임 시작 | 큐 잡기, 출동하기, 랜덤 호출 시작 |
| 시즌 | 이번 시즌 난장판 |
| 관리자 | 서버 관제실 |
| 이벤트 | 오늘의 억까 룰 |
