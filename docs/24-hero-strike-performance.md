# 24. 히어로 스트라이크 모바일 성능 기준

## 목적

히어로 스트라이크의 게임 규칙과 탄막 가독성을 유지하면서 모바일 프레임 드롭과 GC 멈춤을 줄입니다.

성능 문제를 해결할 때 적 수, 피해량, 탄막 패턴을 먼저 줄이지 않습니다. 렌더링 해상도, 장식 효과, 배열 할당과 오디오 생성 비용을 먼저 낮춥니다.

## 렌더링 품질

- 기본 내부 렌더 배율 상한: `1.25`
- 저사양 기기 초기 배율 상한: `1.0`
- 평균 프레임 시간이 약 `23ms` 이상으로 0.9초 지속되면 경량 모드로 전환
- 평균 프레임 시간이 약 `18ms` 이하로 4초 유지되면 일반 모드로 복귀
- 저메모리 또는 4코어 이하 기기는 경량 모드를 유지

경량 모드에서만 줄어드는 요소:

- 탄환 외곽 광택
- 별 렌더 밀도
- 버스트 파티클 생성량
- 내부 픽셀 배율

게임 판정, 적 수, 적탄 패턴, 플레이어 탄환 수와 피해량은 바뀌지 않습니다.

## 런타임 상한

- 전체 탄환: 220
- 미사일: 16
- 파티클: 96
- 플로팅 텍스트: 8

탄환 상한은 전투 규칙이므로 이번 최적화에서 낮추지 않습니다. 파티클과 텍스트는 장식 요소이므로 모바일 안정성을 위해 축소합니다.

## 할당 최소화

다음 배열은 매 프레임 새 배열을 만들지 않고 제자리 압축합니다.

- 탄환
- 적
- 파티클
- 플로팅 텍스트
- 픽업

효과음 노이즈는 길이별 `AudioBuffer`를 재사용하고, 종료된 오디오 노드는 즉시 연결 해제합니다.

## 렌더러 원칙

- 탄환마다 `save / translate / rotate / restore`를 반복하지 않습니다.
- 탄환 방향 벡터로 화면 좌표를 직접 계산합니다.
- 정적 배경은 스테이지와 오버드라이브 상태별로 캐시합니다.
- 엔티티, 탄환과 이펙트 렌더러를 분리합니다.
- 경량 모드에서도 적탄과 경험치의 색상·형태 구분은 유지합니다.

## 모바일 회귀 테스트

1. 펄스 리피터와 산탄 강화 상태에서 4웨이브 이상 플레이합니다.
2. 적탄이 화면 상한에 가까운 보스 3페이즈를 확인합니다.
3. FLOW 발동과 폭발·연쇄·미사일이 동시에 발생하는 상황을 확인합니다.
4. 궁극기와 화면 정리 폭탄 직후 긴 멈춤이 없는지 확인합니다.
5. 앱 전환 후 복귀했을 때 큰 `dt`가 들어가지 않는지 확인합니다.
6. 경량 모드 전환 시 캔버스 크기나 터치 좌표가 어긋나지 않는지 확인합니다.
7. 적탄, 경험치, 플레이어 탄환의 가독성이 유지되는지 확인합니다.
8. 효과음이 누적되거나 지연 재생되지 않는지 확인합니다.

## 추가 계측 후보

실기기 플레이 데이터가 확보되면 개발 모드에만 다음 값을 표시할 수 있습니다.

- 평균 프레임 시간
- 현재 품질 단계
- 탄환·파티클·적 개수
- 업데이트와 렌더링 각각의 소요 시간

사용자 화면에는 기본적으로 성능 숫자를 노출하지 않습니다.
