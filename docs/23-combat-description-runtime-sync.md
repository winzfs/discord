# 23. 전투 설명 / 실제 런타임 싱크 점검

## 1. 목적

영웅 카드 설명과 인게임 전투 로직이 서로 다르게 보이는 문제를 점검했습니다.

특히 스킬 발동조건이 `기본 공격 시 n% 확률`로 표시되는데, 실제 전투에서도 같은 기준으로 발동하는지 확인했습니다.

## 2. 신화 영웅 스킬 확인

신화 영웅 스킬은 이미 확률 기준으로 동작하고 있었습니다.

적용 파일:

```text
apps/web/src/game-client/pixi/pixiSkillRuntime.ts
```

실제 기준:

```text
공격 스킬: 42%
제어 스킬: 30%
지원 스킬: 24%
겐지 질풍참: 22%
자리야 방벽 충전: 차지 3 이상 + 24%
자리야 입자포: 확률 스킬이 아니라 지속 강화 공격타입
```

카드 설명도 이 기준에 맞춰져 있습니다.

## 3. 비신화/기본 영웅 스킬 수정

기존 문제:

```text
카드 설명: 기본 공격 시 n% 확률
실제 로직: 매 공격마다 스킬 효과 적용
```

수정 후 실제 로직:

```text
공격/피해/광역/연쇄 계열: 42%
제어/표식/취약 계열: 30%
지원/경제/소환/방어 계열: 24%
```

적용 파일:

```text
apps/web/src/game-client/pixi/pixiBaseHeroSkillRuntime.ts
apps/web/src/game-client/pixi/pixiCombatRuntime.ts
```

## 4. 이펙트 표시 싱크

비신화 기본 스킬은 이제 실제로 발동했을 때만 다음 요소가 적용됩니다.

```text
피해 배율
광역 피해
연쇄 추가타
제어 효과
경제 보상
스킬 이펙트
스킬 팝업
```

기존에는 스킬 이펙트가 실제 발동 여부와 무관하게 보일 수 있었으나, 이제 발동 플래그를 기준으로 표시됩니다.

## 5. 자리야 지속 강화 확인

자리야는 다음 로직이 실제로 존재합니다.

```text
같은 대상을 1.2초 안에 계속 공격하면 차지 증가
최대 6단계
피해 배율 = 0.78 + charge × 0.16
```

따라서 카드의 `공격타입: 지속 강화` 설명과 실제 전투가 일치합니다.

## 6. 확인 필요

```bash
pnpm typecheck
pnpm build:web
```

테스트 체크리스트:

- 비신화 영웅 스킬이 매 공격마다 발동하지 않는지
- 공격 계열 스킬 팝업이 체감상 약 42% 빈도로 뜨는지
- 제어 계열 스킬 팝업이 체감상 약 30% 빈도로 뜨는지
- 지원/경제 계열 스킬 팝업이 체감상 약 24% 빈도로 뜨는지
- 스킬 이펙트가 실제 발동 시에만 보이는지
- 자리야가 같은 대상을 계속 공격할수록 데미지가 증가하는지
