# 13. 코인 이자 시스템

## 1. 목적

코인 이자 시스템은 전투 중 재화를 즉시 소비할지, 일부러 모아서 다음 웨이브 보너스를 받을지 선택하게 만드는 경제 장치입니다.

기대 효과:

- 소환/강화 버튼을 무조건 누르는 플레이를 줄입니다.
- 코인을 아껴 다음 웨이브 이자를 받는 전략이 생깁니다.
- 초반에는 이자보다 전투력이 중요하고, 중후반에는 경제 관리가 중요해집니다.

## 2. 현재 적용 방식

적용 위치:

```text
apps/web/src/game-client/pixi/pixiCoinInterestRuntime.ts
apps/web/src/game-client/pixi/pixiWaveFlowRuntime.ts
apps/web/src/game-client/pixi/pixiWaveFeedbackRuntime.ts
apps/web/src/game-client/pixi/pixiGameTypes.ts
```

웨이브가 정상 종료될 때 현재 보유 코인을 기준으로 이자를 계산합니다.

현재 규칙:

```text
최소 보유 코인: 30
이자율: 8%
웨이브당 최대 이자: 30 코인
실패 상태에서는 이자 지급 없음
```

## 3. 계산 규칙

```text
이자 = floor(현재 코인 * 0.08)
최종 이자 = min(이자, 30)
```

단, 현재 코인이 30 미만이면 이자는 0입니다.

예시:

| 현재 코인 | 계산 이자 | 실제 지급 |
|---:|---:|---:|
| 20 | 0 | 0 |
| 30 | 2 | 2 |
| 100 | 8 | 8 |
| 250 | 20 | 20 |
| 500 | 40 | 30 |

## 4. 결과 표시

이자를 받은 경우 웨이브 결과 중앙에 별도 노란색 플로팅 텍스트로 표시합니다.

예시:

```text
코인 이자 +8 (8%)
```

이자가 한도에 걸린 경우:

```text
코인 이자 +30 (8% 한도)
```

처치 보상과 행운석 보상은 별도 줄로 표시합니다.

```text
처치 보상 +48  행운석 +1
```

이전에는 이자 문구가 처치 보상 줄 뒤에 붙어 보상 메뉴에 가려질 수 있었기 때문에, 현재는 이자 표시를 독립 플로팅 텍스트로 분리했습니다.

## 5. 밸런스 의도

현재 값은 MVP 기준입니다.

- 최소 30코인: 초반에 무조건 이자가 발생하지 않게 하기 위함
- 8%: 체감은 되지만 소환보다 압도적으로 좋지 않게 하기 위함
- 최대 30코인: 과도한 저축 플레이가 정답이 되지 않게 하기 위함

## 6. 다음 개선 후보

- HUD에 다음 이자 예상치 표시
- 웨이브 보상 선택지로 이자율 일시 증가 추가
- 경제형 영웅/유물이 이자 상한을 올리도록 확장
- 보스 웨이브 전 이자 보너스 이벤트 추가
- 이자 지급 시 별도 금화 파티클 연출 추가

## 7. 테스트 체크리스트

- 30코인 미만에서는 이자가 0인지 확인
- 30코인 이상에서 웨이브 종료 시 이자가 지급되는지 확인
- 500코인 이상에서 이자가 30으로 제한되는지 확인
- 실패 상태에서는 이자가 지급되지 않는지 확인
- 이자를 받은 경우 `코인 이자 +N` 노란색 플로팅 텍스트가 중앙에 표시되는지 확인
- 웨이브 보상 메뉴가 떠도 이자 표시가 가려지지 않는지 확인
- 이자 지급 후 HUD 코인이 증가하는지 확인
