# 운빨존많겜류 시스템 맞춤 반영안

> 목적: 원작을 복제하지 않고, 랜덤 소환 + 3중첩 + 이동 배치 + 합성 + 웨이브/보스 압박 구조를 Discord 웹게임 `discord-random-defense`에 맞게 재설계한다.

## 1. 플레이 터치/클릭 흐름

### 로비

| 터치 대상 | 동작 | 우리 게임 반영 |
| --- | --- | --- |
| 하단 `상점` | 재화/조각 구매 화면 | 임시 상점 유지, 추후 영웅 조각/유물 조각 상품 연결 |
| 하단 `영웅` | 전체 영웅 도감 | 모든 `packages/game` 영웅 표시. 보유 영웅만 활성화, 미보유 영웅은 회색 잠금 카드 |
| 영웅 카드 | 상세 패널 열기 | 보유 영웅만 클릭 가능. 전투력/공속/사거리/태그/조각/레벨 표시 |
| 영웅 업그레이드 | 조각 + 골드 소모 | 조각 요구량 `5 + level * 5`, 비용 `650 + level * 350` |
| 하단 `유물` | 전체 유물 도감 | 모든 유물 표시. 보유 유물만 활성화, 미보유 유물은 회색 잠금 카드 |
| 유물 카드 | 상세 패널 열기 | 보유 유물만 클릭 가능. 효과/현재 수치/레벨당 증가/조각 표시 |
| 유물 업그레이드 | 조각 + 골드 소모 | 조각 요구량 `2 + level * 2`, 비용 `520 + level * 280` |
| `빠른 시작` | 전투 진입 | `/play?difficulty=n`으로 이동 |

### 인게임

| 터치 대상 | 동작 | 우선순위 |
| --- | --- | --- |
| 소환 버튼 | 현재 코인 소모 후 무작위 영웅 소환 | 핵심 |
| 보드 칸 유닛 | 유닛 정보 + 합성/판매 메뉴 | 핵심 |
| 유닛 드래그 | 대상 칸으로 이동, 3중첩 제한 확인 | 핵심 |
| 합성 버튼 | 같은 칸 3중첩이면 상위 등급 변환 | 핵심 |
| 판매 버튼 | 최상단 유닛 판매 후 일부 코인 회수 | 핵심 |
| 신화 버튼 | 신화 조합 메뉴 표시 | 핵심 |
| 신화 조합 행 | 재료가 있으면 신화 영웅 생성 | 핵심 |
| 도박 버튼 | 행운석 소모 후 고등급 소환 시도 | 중간 |
| 공격력 강화 | 코인 소모 후 현재 판 공격력 증가 | 핵심 |
| 웨이브 시작 | 카운트다운 생략 후 즉시 전투 | 핵심 |

## 2. 로비 수집/성장 설계

### 영웅

- 전체 영웅은 `packages/game/src/data/heroes.ts`의 `heroes` 배열을 기준으로 표시한다.
- 카드 상태는 `owned` 기준으로 나눈다.
  - 보유: 컬러 카드, 상세 열기 가능, 레벨/조각/역할 표시
  - 미보유: 회색 카드, 클릭 비활성, `?` 초상화, `비활성` 표시
- 레벨업은 보유 영웅만 가능하다.
- 레벨당 전투력은 `basePower * (1 + (level - 1) * 0.12)`로 표시한다.
- 조각 요구량은 `min(120, 5 + level * 5)`.
- 골드 비용은 `650 + level * 350`.

### 유물

- 전체 유물은 `packages/game/src/data/artifacts.ts`의 `artifacts` 배열을 기준으로 표시한다.
- 카드 상태는 `owned` 기준으로 나눈다.
  - 보유: 컬러 카드, 상세 열기 가능, 레벨/조각/효과 표시
  - 미보유: 회색 카드, 클릭 비활성, `?` 아이콘, `비활성` 표시
- 레벨업은 보유 유물만 가능하다.
- 현재 효과는 `baseEffect + (level - 1) * effectPerLevel`.
- 조각 요구량은 `min(80, 2 + level * 2)`.
- 골드 비용은 `520 + level * 280`.

## 3. 전투 핵심 루프

1. 기본 코인과 행운석 지급
2. 소환 버튼으로 무작위 영웅 생성
3. 빈 슬롯이 있는 보드 칸에 자동 배치
4. 플레이어가 유닛을 직접 이동해 핵심 위치 정리
5. 한 칸에 최대 3마리까지 중첩
6. 3중첩 또는 조합 재료로 합성/신화 조합
7. 웨이브 중 자동 공격
8. 몬스터 처치로 코인 획득
9. 코인을 소환/공격력 강화/운영에 재투자
10. 5웨이브마다 보스 체크
11. 30웨이브 클리어 또는 코어 HP 소진으로 종료

## 4. 우리 게임용 밸런스 기준

### 소환 확률

| 등급 | 비중 | 역할 |
| --- | ---: | --- |
| 일반 | 72 | 초반 방어, 합성 재료 |
| 희귀 | 22 | 중반 안정화, 조합 재료 |
| 영웅 | 5 | 주력 딜/보조 |
| 전설 | 1 | 핵심 빌드 |
| 신화 | 직접 소환 제외 | 조합/특수 경로로만 획득 |

신화는 일반 소환에서 직접 나오지 않게 유지한다. 대신 신화 조합 메뉴와 행운석/이벤트 경로로 얻는 것이 목표성을 만든다.

### 몬스터/웨이브 산식

- 일반 웨이브 기본 수량: `6 + floor(wave * 1.55)`
- 일반 웨이브 보상: `8 + wave * 2`
- 보스 웨이브 주기: 5웨이브마다 1회
- 보스 웨이브 보상: `40 + wave * 4`
- 전투 시간: 일반/보스 모두 14초 기준
- 보스는 단일 화력과 제어 능력을 검사하는 역할이다.

### 몬스터 역할

| 몬스터 | 역할 | 현재 기준 |
| --- | --- | ---: |
| 잡버그 | 기본 잡몹 | HP 42 / 보상 4 |
| 핑러너 | 빠른 누수 압박 | HP 34 / 보상 5 |
| 렉덩어리 | 느린 탱커 | HP 120 / 보상 11 |
| 엘리트버그 | 중반 압박 | HP 210 / 보상 22 |
| 서버크래셔 | 보스 | HP 620 / 보상 85 |

## 5. 구현 우선순위

1. 로비 전체 영웅/유물 표시와 잠금 상태
2. 로비 레벨업 비용/조각 소모
3. 인게임 유닛 선택/메뉴/신화 메뉴 터치 안정화
4. 웨이브별 몬스터 수/보상 산식 정리
5. 영웅 레벨/유물 효과를 실제 인게임 전투력에 반영
6. 보스 패턴과 전용 경고 연출 강화
7. Discord 계정 저장/서버 랭킹/협동 구조 확장

## 6. 주의사항

- 원작 수치를 그대로 따라가지 않는다.
- 원작의 구체 캐릭터, 명칭, 이미지, 고유 조합식을 복제하지 않는다.
- 참고하는 것은 장르 구조와 조작 흐름이다.
- 우리 게임은 Discord 커뮤니티 연동, 서버 랭킹, 빠른 웹 플레이에 맞춰 더 짧고 명확한 루프로 설계한다.
