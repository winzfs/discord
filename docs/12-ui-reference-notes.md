# 12. UI 참고 스크린샷 분석 노트

## 1. 목적

사용자가 추가로 제공한 모바일 캐주얼 디펜스/수집형 게임 스크린샷을 기준으로, 현재 `/play` PixiJS 게임 화면과 향후 웹 대시보드/메타 성장 화면의 UI 방향을 정리합니다.

이 문서는 특정 게임을 그대로 복제하기 위한 문서가 아니라, 모바일 세로형 디펜스 게임에서 사용성이 검증된 UI 패턴을 우리 프로젝트에 맞게 재해석하기 위한 기준입니다.

## 2. 전체 UI 방향

현재 프로젝트는 다음 방향으로 조정합니다.

- 모바일 세로 화면 우선
- 큰 버튼, 굵은 외곽선, 높은 명도 대비
- 중앙 전투 보드와 외곽 몬스터 경로
- 상단은 현재 상태 정보, 하단은 핵심 조작 버튼
- 팝업은 두꺼운 프레임과 종이/나무 패널 느낌
- 상점, 영웅, 유물, 퀘스트는 카드 그리드 UI로 통일

## 3. 전투 화면 참고 포인트

### 3.1 상단 HUD

참고 스크린샷의 공통 요소:

- 좌상단 플레이어 카드
- 플레이어 닉네임 또는 캐릭터명
- 레벨 표시
- 경험치/진행도 바
- 상단 중앙 웨이브/타이머
- 상단 우측 재화 묶음
- 코어 HP 또는 보스 HP 바

현재 `/play`에 이미 있는 요소:

- WAVE 표시
- 코어 HP 바
- 보유 코인
- 점수
- 전투력
- 유닛 수 `n / 16`

향후 개선 방향:

- 좌상단에 플레이어 카드형 패널 추가
- 웨이브 박스를 더 크게 만들고 타이머 강조
- 코어 HP와 보스 HP를 구분
- 전투력/위협도는 항상 표시보다 웨이브 결과 또는 상세 버튼에서 보여주는 방식 고려

### 3.2 전투 보드

참고 스크린샷의 공통 요소:

- 중앙에 사각형 배치판
- 외곽 길을 따라 적 이동
- 보드 위 유닛은 자동 공격
- 유닛 주변에 사거리 원 또는 공격 이펙트 표시
- 데미지 숫자가 유닛/적 위에 떠오름

현재 `/play` 구조와 잘 맞는 부분:

- 중앙 4x4 보드
- 외곽 흙길 이동 경로
- 몬스터 이동 연출

향후 개선 방향:

- 영웅 역할별 실루엣 구분
  - 딜러: 원형 몸체 + 무기/총구 느낌
  - 탱커: 방패형 몸체
  - 지원: 십자/오라/별 느낌
- 적 타입별 실루엣 구분
  - 기본: 작은 슬라임형
  - 빠름: 작고 길쭉한 러너형
  - 탱커: 둥글고 큰 덩어리형
  - 보스: 큰 몸체 + 별도 HP 바
- 공격 연출은 실시간 전투 이전에도 가짜 투사체/데미지 숫자로 먼저 추가

### 3.3 하단 액션 영역

참고 스크린샷의 핵심 요소:

- 중앙 큰 `소환` 버튼
- 좌측 특수 성장 버튼
- 우측 도박/상점/보조 버튼
- 버튼 하단에 비용 표시
- 버튼이 불가능할 때 어둡게 비활성화
- 무료/광고 보상은 별도 뱃지로 강조

현재 `/play`에 있는 요소:

- 하단 중앙 소환 버튼
- 일반/희귀 합성 버튼
- 웨이브 시작 버튼

즉시 개선할 항목:

- 소환 버튼 비활성 상태 표시
  - 보유 코인 부족
  - 보드 가득 참
  - 게임오버/클리어 상태
- 합성 버튼 상태 표시
  - `일반 0/3`, `일반 1/3`, `일반 2/3`, `일반 가능`
  - `희귀 0/3`, `희귀 1/3`, `희귀 2/3`, `희귀 가능`
- 가능 상태일 때 버튼 색상을 더 밝게 강조

## 4. 상점 화면 참고 포인트

참고 스크린샷의 상점 구조:

- 3열 카드 그리드
- 상품명 상단 표시
- 상품 아이콘 중앙 배치
- 수량 우하단 표시
- 가격 하단 노란 영역
- 할인 라벨을 카드 좌상단에 부착
- 광고 무료 보상 카드
- 초기화/갱신 버튼

우리 프로젝트 재해석:

- 상점은 MVP 이후 메타 성장 화면으로 보류
- 디스코드 활동 보상과 연결 가능
- 예시 상품
  - 영웅 초대장
  - 유물 조각
  - 일일 보상 상자
  - 이벤트 토큰
  - 서버 활동 포인트 교환권

주의:

- 과금 구조는 현재 보류
- 광고 보상은 웹앱에서는 바로 구현하지 않고, 무료 보상 또는 디스코드 활동 보상으로 대체 가능

## 5. 영웅 상세/도감 화면 참고 포인트

참고 스크린샷의 영웅 상세 구조:

- 상단 큰 영웅 아이콘
- 레벨 표시
- 이름
- 등급/역할/타입 태그
- 조합/시너지 표시
- 공격력, 공격속도 같은 핵심 스탯
- 레벨별 해금 효과 목록
- 하단 스킬 아이콘 줄
- 업그레이드 비용과 버튼

우리 프로젝트의 현재 데이터와 연결 가능한 항목:

- `displayName`
- `role`
- `grade`
- `attackType`
- `power`
- `attackSpeed`
- `range`
- `skillIds`
- `tags`

향후 구현 방향:

- `/heroes` 또는 게임 내 도감 팝업으로 구현
- 먼저 읽기 전용 도감으로 만들고, 이후 업그레이드/조각 시스템 연결
- 영웅 조각은 Discord 활동 보상 또는 게임 보상으로 지급 가능

## 6. 유물/장비 화면 참고 포인트

참고 스크린샷의 유물 화면 구조:

- 카드형 그리드
- 각 카드에 레벨 표시
- 조각 진행도 `n / required`
- 진행도 바
- 업그레이드 가능 시 초록 진행도 강조
- 탭으로 영웅/신화/유물 전환

우리 프로젝트 재해석:

- 유물은 MVP 이후 확장
- 전투 중 랜덤 선택 유물과 영구 수집 유물을 분리할 수 있음
- MVP 이후 추천 구조
  - 전투 중 유물: 해당 런에서만 적용
  - 계정 유물: 영구 성장, 랭킹/시즌과 연결

## 7. 퀘스트 화면 참고 포인트

참고 스크린샷의 퀘스트 구조:

- 일일 퀘스트/업적 탭
- 리스트형 항목
- 보상 아이콘
- 진행도 바
- 완료 보상 버튼

우리 프로젝트 재해석:

- 디스코드 커뮤니티와 연결하기 좋음
- 예시 퀘스트
  - 오늘 게임 1회 플레이
  - 웨이브 10 도달
  - 일반 합성 3회
  - 디스코드 서버 출석
  - 음성 채널 참여
  - 커뮤니티 글 작성

MVP에서는 게임 내부 퀘스트만 먼저 고려합니다.

## 8. 영웅 모집/초대장 화면 참고 포인트

참고 스크린샷의 모집 구조:

- 큰 제목 배너
- 중앙 설명 영역
- 모집 진행도 `0/5`
- 일반 모집/10회 모집 버튼
- 초대장 비용 표시
- 특정 조건 달성 시 무료 모집 보상

우리 프로젝트 재해석:

- 현재 소환은 전투 중 랜덤 유닛 소환
- 메타 모집은 별도 시스템으로 분리 가능
- 초대장/영웅 조각은 로그인/DB/랭킹 연결 이후 구현

## 9. 현재 코드에 바로 반영할 우선순위

### 9.1 즉시 반영

1. 소환 버튼 비활성 상태 표시
2. 합성 가능 여부 UI 표시
3. 하단 버튼을 모바일 게임식 큰 카드 버튼으로 정리
4. 역할별 유닛 실루엣 구분
5. 적 타입별 실루엣 구분

### 9.2 다음 반영

1. 웨이브 결과 팝업
2. 데미지 숫자 연출
3. 영웅 공격 모션
4. 보스 HP 바
5. 보스 웨이브 경고 배너

### 9.3 이후 반영

1. 영웅 도감
2. 유물 도감
3. 퀘스트
4. 영웅 모집
5. 상점/보상
6. Discord OAuth 기반 계정 저장

## 10. 시각 톤 가이드

- 배경: 밝은 숲/필드, 갈색 UI 패널
- 버튼: 노란색, 주황색 중심
- 비활성 버튼: 어두운 회갈색 + 낮은 알파
- 강조: 초록색 화살표, 노란 반짝임, 빨간 할인/경고 라벨
- 텍스트: 굵은 흰색 글자 + 검정 외곽선
- 카드: 둥근 모서리, 두꺼운 외곽선, 그림자
- 팝업: 어두운 반투명 배경 + 종이/나무 패널

## 11. 주의사항

- 참고 스크린샷의 UI를 그대로 복제하지 않습니다.
- 레이아웃, 정보 구조, 버튼 크기, 피드백 방식만 재해석합니다.
- 오버워치 팬게임 성격을 유지하되, 게임 규칙과 UI는 독자적인 커뮤니티 미니게임으로 설계합니다.
- 모바일 웹에서 한 손 조작이 가능해야 합니다.
