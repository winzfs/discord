# 19. 인게임 전술칸 명칭 / 웨이브 결과 피드백 정리

## 1. 목적

현재 전술 타입/웨이브/추천 효과 체계에 맞게 인게임 선택 정보의 셀명을 정리하고, 웨이브 종료 문구가 너무 빠르게 사라지는 문제를 수정했습니다.

## 2. 적용 파일

```text
apps/web/src/game-client/pixi/pixiSelectionRuntime.ts
apps/web/src/game-client/pixi/pixiUnitInfoView.ts
apps/web/src/game-client/pixi/pixiFloatingTextView.ts
```

## 3. 전술칸 명칭 변경

기존 선택 정보 패널에는 다음처럼 표시됐습니다.

```text
셀 1 · 중첩 1/3
```

현재 체계에서는 단순 셀 번호보다 배치 의미가 중요하므로, 보드 행/열을 기준으로 전술칸 이름을 표시하도록 변경했습니다.

예시:

```text
최전방 좌측 전술칸 · 중첩 1/3
전방 중좌 전술칸 · 중첩 2/3
중앙 중우 전술칸 · 중첩 1/3
최후방 우측 전술칸 · 중첩 3/3
```

보드 기준:

```text
5행 × 4열
행: 최전방 / 전방 / 중앙 / 후방 / 최후방
열: 좌측 / 중좌 / 중우 / 우측
```

## 4. 웨이브 종료 문구 유지 시간 증가

기존 모든 플로팅 텍스트는 250ms만 유지되었습니다.

그래서 웨이브 종료 시 다음 문구가 너무 빨리 사라졌습니다.

```text
완벽 방어!
누수 n마리 -n HP
처치 n / 누수 n
코인 이자 +n
처치 보상 +n
```

이제 웨이브 결과 문구만 2200ms 유지되도록 변경했습니다.

일반 피해 숫자나 짧은 전투 팝업은 기존처럼 250ms를 유지합니다.

## 5. 표시 방식

웨이브 결과 문구는 전체 시간 중 약 72% 동안 선명하게 유지되고, 마지막 구간에서 천천히 사라집니다.

목표:

- 보상/누수/완벽 방어 여부를 사용자가 충분히 읽을 수 있게 함
- 일반 전투 숫자 팝업은 기존 속도를 유지해 화면이 지저분해지지 않게 함

## 6. 확인 필요

```bash
pnpm typecheck
pnpm build:web
```

테스트 체크리스트:

- 유닛 선택 시 `셀 1` 대신 전술칸 명칭이 보이는지
- 각 위치별 전방/중앙/후방 이름이 맞는지
- 웨이브 종료 후 결과 문구가 약 2초 이상 유지되는지
- 일반 피해 숫자 팝업이 너무 오래 남지 않는지
