# 24. 스택 유닛 개별 공격 처리

## 1. 목적

한 칸에 같은 유닛이 최대 3개까지 겹칠 수 있으므로, 겹친 유닛도 각각 독립적으로 공격해야 합니다.

이번 기준:

```text
한 칸에 3개 유닛이 있으면 3개 모두 공격
각 유닛은 자기 instanceId 기준으로 공격
각 유닛은 자기 공격력으로 피해 적용
각 유닛은 자기 스킬을 따로 발동
궁극기 충전/자리야 빔 충전도 유닛별로 따로 관리
```

## 2. 적용 파일

```text
apps/web/src/game-client/pixi/pixiCombatRuntime.ts
```

## 3. 기존 문제

보드 시스템의 `getAllBoardHeroes()`는 이미 `cell.units` 전체를 반환합니다.

하지만 전투 런타임에서는 다음 문제가 있었습니다.

```text
MAX_ATTACKERS_PER_TICK 제한으로 일부 유닛만 공격 가능
같은 칸 유닛은 같은 좌표에서 공격해서 한 발처럼 보임
```

## 4. 변경 내용

### 4.1 공격자 제한 제거

기존:

```ts
heroes.slice(0, Math.min(heroes.length, MAX_ATTACKERS_PER_TICK)).forEach(...)
```

변경:

```ts
heroes.forEach(...)
```

이제 보드 위 모든 유닛이 공격 루프에 들어갑니다.

### 4.2 스택별 발사 위치 분리

같은 칸에 겹친 유닛은 공격 시작 좌표를 살짝 다르게 잡습니다.

```text
1번째 유닛: 중앙 위쪽
2번째 유닛: 왼쪽 아래
3번째 유닛: 오른쪽 아래
```

이렇게 해서 3개가 공격해도 한 발처럼 보이지 않고, 서로 다른 공격으로 보입니다.

### 4.3 유닛별 독립 처리 유지

이미 전투 런타임은 `hero.instanceId`를 기준으로 다음 상태를 관리합니다.

```text
heroSpriteAttacks
zaryaBeamCharges
mythicUltimateCharges
```

따라서 같은 영웅 3개가 겹쳐 있어도 각 유닛은 자기 instanceId 기준으로 공격/스킬/궁극기를 처리합니다.

## 5. 테스트 체크리스트

- 같은 유닛 3개를 한 칸에 겹쳤을 때 3개 모두 공격하는지 확인
- 각 공격의 투사체/빔 시작 위치가 살짝 다르게 보이는지 확인
- 각 유닛의 스킬 이펙트가 따로 나가는지 확인
- 자리야 3개가 겹쳤을 때 빔 충전이 각 instanceId별로 따로 동작하는지 확인
- 신화 유닛 3개가 겹쳤을 때 궁극기 충전/발동이 각 유닛별로 동작하는지 확인
- 보드에 유닛이 10개를 넘어도 뒤쪽 유닛이 공격에서 빠지지 않는지 확인
