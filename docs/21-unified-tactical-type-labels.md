# 21. 영웅/공격/스킬 타입 라벨 통일

## 1. 목적

영웅 전술 타입, 공격 타입, 스킬 타입이 비슷한 단어를 중복해서 사용하면서 카드 설명, 인게임 정보, 전투 팝업, 웨이브 추천 UI가 서로 다르게 보이는 문제가 있었습니다.

이번 작업은 실제 시스템을 크게 바꾸는 것이 아니라, 플레이어에게 보이는 타입 라벨과 설명을 하나의 기준으로 통일하는 작업입니다.

## 2. 타입 역할 구분

### 2.1 영웅 전술 타입

캐릭터의 전체 역할입니다.

예시:

```text
정밀: 보스나 고체력 적 처리
광역: 군집 웨이브 처리
제어: 감속/빙결/방벽으로 전선 지연
증폭: 표식/취약/피해 증가
지원: 공격속도/궁극기/전투 흐름 보조
성장: 보상/코인/장기 성장
처형: 체력이 낮은 적 마무리
```

### 2.2 공격 타입

평타가 어떤 방식으로 작동하는지 나타냅니다.

예시:

```text
단일: 한 대상을 안정적으로 공격
범위: 여러 적이나 주변 적에게 피해
제어: 공격과 함께 감속/방해 효과
보조: 직접 피해보다 팀 보조 중심
```

### 2.3 스킬 타입

스킬이 실제로 제공하는 효과입니다.

예시:

```text
피해: 기본 피해량 증가
광역: 몰린 적 처리
연쇄: 추가타/전이 피해
제어: 감속/빙결/방해
증폭: 표식/취약/피해 증가
지원: 공격속도/궁극기 흐름 보조
성장: 보상/코인/장기 성장
처형: 약해진 적 마무리
방어: 방벽/피해 감소/지연
소환: 포탑/보조 화력
```

## 3. 적용 파일

통합 라벨 정의:

```text
packages/game/src/data/tacticalLabels.ts
```

패키지 export:

```text
packages/game/src/index.ts
```

인게임 스킬 설명:

```text
apps/web/src/game-client/pixi/pixiSkillDescriptionRuntime.ts
```

인게임 유닛 정보 패널:

```text
apps/web/src/game-client/pixi/pixiUnitInfoView.ts
```

로비 카드/상세 스킬 설명:

```text
apps/web/src/components/lobby/lobbyHeroSkillDetails.ts
```

전투 중 효과 팝업:

```text
apps/web/src/game-client/pixi/pixiSkillEffectLogRuntime.ts
```

웨이브 추천 HUD:

```text
apps/web/src/game-client/pixi/pixiHudView.ts
```

## 4. 표시 통일 결과

### 4.1 카드 상세

기존에는 태그 기반 설명과 효과 타입 배지가 섞여 있었습니다.

이제 카드 상세의 스킬 배지는 `getSkillEffectLabel()` 기준으로 표시됩니다.

### 4.2 인게임 유닛 정보

스킬 카드 제목이 기존 `공격/제어/지원` 중심에서 실제 효과 타입 중심으로 바뀝니다.

예시:

```text
펄스 폭탄 [광역·게이지]
수리검 [연쇄·고유]
수면총 [제어·고유]
나노 강화제 [증폭·게이지]
```

### 4.3 전투 팝업

전투 중 효과 팝업도 같은 라벨을 사용합니다.

```text
광역
연쇄
제어
증폭
지원
성장
처형
방어
소환
```

### 4.4 웨이브 추천 HUD

웨이브 추천 타입도 같은 라벨을 사용합니다.

```text
추천 제어 · 처형 · 연쇄
추천 증폭 · 피해 · 처형
```

## 5. 확인 필요

```bash
pnpm typecheck
pnpm build:web
```

테스트 체크리스트:

- 로비 카드 상세의 스킬 타입 배지가 통일 라벨로 보이는지
- 인게임 유닛 정보 패널의 스킬 제목이 `[광역·게이지]` 같은 형태로 보이는지
- 전투 중 팝업이 같은 단어로 뜨는지
- 웨이브 추천 HUD가 같은 단어를 쓰는지
- 기존 영웅 공격/스킬 동작이 깨지지 않는지
