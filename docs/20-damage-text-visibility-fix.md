# 20. 데미지 숫자 표시 누락 수정

## 1. 문제

캐릭터가 공격해도 데미지 숫자가 표시되지 않는 경우가 있었습니다.

특히 옆에 다른 캐릭터를 배치했을 때 발생했습니다.

## 2. 원인

전투 화면 도배를 막기 위해 일부 데미지 표시가 다음 조건에 묶여 있었습니다.

```ts
if (index === 0) options.floatText(...)
```

`index`는 보드 전체 영웅 순서 기준입니다.

그래서 공격한 영웅이 보드 순서상 첫 번째가 아니면 실제로 공격이 들어가도 데미지 숫자가 뜨지 않았습니다.

## 3. 수정 내용

다음 공격 타입의 데미지 표시에서 `index === 0` 조건을 제거했습니다.

```text
기본 투사체 공격
윈스턴 테슬라 공격
자리야 빔 공격
```

이제 실제 공격이 들어간 경우에는 보드 배치 순서와 상관없이 데미지 숫자가 표시됩니다.

## 4. 적용 파일

```text
apps/web/src/game-client/pixi/pixiCombatRuntime.ts
```

## 5. 확인 필요

```bash
pnpm typecheck
pnpm build:web
```

테스트 체크리스트:

- 캐릭터 1명만 배치했을 때 데미지 숫자가 뜨는지
- 옆 칸에 다른 캐릭터를 배치해도 기존 캐릭터 데미지 숫자가 계속 뜨는지
- 윈스턴/자리야도 배치 순서와 상관없이 데미지 숫자가 뜨는지
- 데미지 숫자가 너무 과하게 도배되지 않는지
