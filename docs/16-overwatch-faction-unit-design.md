# 16. 오버워치 세력 기반 유닛 구성안

## 1. 문서 목적

이 문서는 `docs/15-overwatch-faction-bible.md`의 세력 해석을 실제 랜덤 디펜스 유닛 풀로 변환하기 위한 설계 문서입니다.

목표:

- 일반/희귀/영웅 등급 유닛을 세계관 기반 병과로 전환합니다.
- 각 유닛이 어느 세력에 속하는지 명확히 합니다.
- 모든 유닛에 짧은 배경설명을 부여합니다.
- 탱커/딜러/지원 역할과 스킬 태그가 세력 정체성과 연결되도록 합니다.
- 신화 영웅 조합 재료, 패스 보상, 세력별 이벤트까지 확장 가능한 구조를 만듭니다.

## 2. 유닛 데이터 설계 원칙

현재 코드 구조는 다음 데이터를 기준으로 유닛을 구성합니다.

```ts
id: string;
displayName: string;
description: string;
role: 'tank' | 'damage' | 'support';
grade: 'common' | 'rare' | 'epic' | 'legendary' | 'mythic';
attackType: 'single' | 'area' | 'support' | 'control';
skillIds: string[];
tags: string[];
```

중요:

- `id`는 저장 데이터, 조합, 스킬 연결에 영향을 줄 수 있으므로 가능한 유지합니다.
- 표시명, 설명, 태그는 세계관에 맞게 바꿔도 안전합니다.
- 세계관 세력은 `tags`에 넣어 추후 시너지로 확장합니다.
- 세력명만 붙이지 말고 전투 기능과 연결합니다.
- 로비 상세와 인게임 유닛 정보 패널은 `description`을 표시합니다.

## 6. 모든 유닛 배경설명 적용

`HeroDefinition`에 `description` 필드를 추가했습니다.

표시 위치:

```text
apps/web/src/pages/LobbyPage.tsx
apps/web/src/game-client/pixi/pixiUnitInfoView.ts
```

- 로비 영웅 상세: 전체 배경설명을 표시합니다.
- 인게임 유닛 정보 패널: 짧게 줄인 배경설명을 표시합니다.

## 9. 신화 조합 연결안

신화 조합은 공식 영웅 감성과 하위 세력 병과를 연결하면 깊이감이 생깁니다.

| 신화 후보 | 재료 방향 | 설명 |
|---|---|---|
| 아나 | 카이로 나노 보조병 + 야전 의무관 + 연구소 지원 유닛 | 생체/나노 지원 계열 |
| 키리코 | 요괴 자경단원 + 카네자카 감시꾼 + 하시모토 관련 유닛 | 카네자카 서사 연결 |
| 겐지 | 하시모토/시마다 계열 + 빠른 딜러 + 표식 유닛 | 암살/검술 계열 |
| 윈스턴 | 달 기지 중력 기술자 + 연구소 유닛 + 전기/체인 유닛 | 과학/테슬라 계열 |
| D.Va | 파일럿 후보생 + 메카 정비공 + 범위 딜러 | 메카/방어 매트릭스 계열 |
| 자리야 | 헬릭스/군사 탱커 + 방벽 유닛 + 중력 실험체 | 방벽/중력 제어 계열 |
| 캐서디 | 탈론 추적요원 + 정커 총잡이 + 보안 저격수 | 총잡이/표식/보스딜 계열 |
| 일리아리 | 태양 연구원 + 지원 포탑 유닛 + 폭발/표식 유닛 | 태양 에너지/표식 폭발 계열 |

## 10. 다음 구현 우선순위

1. 하시모토/비슈카르/눌 섹터 계열 신규 유닛을 더 추가합니다.
2. 세력 태그별 보너스를 구현합니다.
3. 신화 조합 메뉴에서 세력 재료 조건을 표시합니다.
4. 패스 보상에 세력별 조각 상자를 추가합니다.
5. 웨이브 테마에 세력 침공 이름을 붙입니다.
6. 영웅 정보 패널에 세력명을 별도 라벨로 표시합니다.

## 11. 개발 메모

세계관 깊이를 올리는 핵심은 유닛 이름만 바꾸는 것이 아니라, 다음을 함께 묶는 것입니다.

```text
세력 → 병과 → 전투 태그 → 배치 전략 → 조합 재료 → 패스 보상 → 웨이브 테마
```

이 연결이 생기면 일반/희귀/영웅 유닛도 단순 합성 재료가 아니라, 판의 방향을 정하는 선택지가 됩니다.
