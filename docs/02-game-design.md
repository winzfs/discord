# 02. 게임 기획서

## 1. 장르 정의

장르: **랜덤 협동 디펜스 웹게임**

핵심 키워드:

- 랜덤 소환
- 웨이브 방어
- 유닛 합성
- 등급 성장
- 짧은 판당 플레이
- 운빨과 빌드 선택
- 서버 랭킹
- 디스코드 커뮤니티 연동

## 2. 한 줄 콘셉트

디스코드 서버를 침공하는 버그 몬스터를 막기 위해, 랜덤으로 호출되는 수호대를 배치하고 합성하며 웨이브를 버티는 커뮤니티용 웹 디펜스 게임입니다.

## 3. 플레이 목표

플레이어는 제한된 필드에 랜덤 유닛을 소환하고, 같은 등급의 유닛을 합성하여 더 강한 유닛을 만들며, 몰려오는 웨이브를 최대한 오래 막습니다.

승리/목표 구조:

- 기본 모드: 정해진 웨이브까지 생존하면 클리어
- 무한 모드: 최대한 높은 웨이브까지 생존
- 시즌 모드: 기간 내 최고 점수 경쟁
- 이벤트 모드: 특정 조건으로 점수 경쟁

MVP에서는 `기본 웨이브 모드` 또는 `무한 웨이브 모드` 중 하나만 구현합니다.

## 4. 핵심 플레이 루프

1. 게임 시작
2. 초기 자원 지급
3. 랜덤 유닛 호출
4. 적 웨이브 등장
5. 유닛이 자동 공격
6. 적 처치 시 자원 획득
7. 자원으로 추가 호출 또는 강화
8. 같은 조건의 유닛 합성
9. 보스 웨이브 도전
10. 실패 또는 클리어
11. 점수 저장
12. 랭킹 반영

## 5. 한 판의 구성

### 5.1 시작 단계

- 플레이어는 빈 필드에서 시작합니다.
- 기본 자원을 받습니다.
- 첫 유닛을 몇 번 무료로 호출할 수 있습니다.
- 튜토리얼에서는 `호출하기`, `합성하기`, `웨이브 시작`만 안내합니다.

### 5.2 전투 단계

- 적은 정해진 경로를 따라 이동합니다.
- 유닛은 자동으로 공격합니다.
- 플레이어는 전투 중에도 유닛을 호출하거나 합성할 수 있습니다.
- 전투 조작은 모바일에서도 가능한 수준으로 단순해야 합니다.

### 5.3 성장 단계

- 적 처치로 자원을 얻습니다.
- 자원으로 유닛 호출, 전체 공격력 강화, 특정 역할 강화 등을 선택합니다.
- 같은 등급 유닛을 합성하여 상위 등급을 노립니다.

### 5.4 보스 단계

- 일정 웨이브마다 보스가 등장합니다.
- 보스는 체력이 높거나 특수 패턴을 가집니다.
- 보스를 제한 시간 안에 처치하지 못하면 생명력이 크게 감소하거나 게임이 종료됩니다.

### 5.5 종료 단계

게임 종료 시 다음 결과를 보여줍니다.

- 도달 웨이브
- 처치 수
- 보스 처치 수
- 최종 점수
- 획득 보상
- 개인 최고 기록 갱신 여부
- 서버 랭킹 변동

## 6. 필드 구조

MVP 추천 필드:

- 3 x 5 또는 4 x 4 슬롯형 필드
- 유닛은 슬롯에 배치됨
- 적은 고정 경로를 따라 이동
- 플레이어는 유닛 위치 이동을 하지 않거나, 추후 기능으로 추가

초기에는 조작 복잡도를 줄이기 위해 위치 이동을 제외합니다.

대안:

- 자동 배치형: 호출하면 빈 슬롯에 자동 배치
- 선택 배치형: 호출 후 원하는 슬롯 클릭

MVP에서는 자동 배치형을 추천합니다.

## 7. 자원 구조

### 7.1 전투 자원

전투 중 사용하는 자원입니다.

후보명:

- 에너지
- 서버 포인트
- 호출 포인트
- 도파민

사용처:

- 유닛 호출
- 기본 강화
- 특수 스킬 사용

### 7.2 계정 자원

게임 밖에서 누적되는 자원입니다.

후보명:

- 토큰
- 시즌 코인
- 서버 칩

사용처:

- 유물 해금
- 마스코트 해금
- 프로필 꾸미기
- 시즌 보상 교환

MVP에서는 계정 자원은 기록만 남기고 사용처는 나중에 열어도 됩니다.

## 8. 유닛 역할

오버워치 감성을 직접 사용하지 않고, 역할 구조만 가져옵니다.

역할 후보:

| 역할 | 설명 |
|---|---|
| 공격형 | 단일 대상 또는 범위 피해 담당 |
| 방어형 | 적을 느리게 하거나 길목을 버티는 역할 |
| 지원형 | 아군 공격 속도, 치명타, 자원 획득 보조 |
| 제어형 | 기절, 둔화, 밀치기 등 군중 제어 |
| 행운형 | 확률 기반 효과, 보상 증가, 랜덤 강화 |

MVP에서는 공격형, 지원형, 제어형만 있어도 충분합니다.

## 9. 유닛 등급

추천 등급:

1. Common / 일반
2. Rare / 특수
3. Epic / 에이스
4. Legendary / 전설
5. Mythic / 비밀병기

초기 MVP에서는 4등급까지만 구현해도 됩니다.

등급별 특징:

- 일반: 자주 등장, 합성 재료
- 특수: 기본 전투력 담당
- 에이스: 빌드 중심
- 전설: 판의 방향을 바꾸는 핵심 유닛
- 비밀병기: 낮은 확률, 이벤트성 연출

## 10. 적 구성

적은 원작과 무관한 오리지널 개념으로 구성합니다.

적 후보:

| 타입 | 특징 |
|---|---|
| 잡버그 | 기본 적, 낮은 체력 |
| 빠른버그 | 이동 속도 빠름 |
| 튼튼버그 | 체력 높음 |
| 방해버그 | 일정 시간 유닛 공격 방해 |
| 분열버그 | 죽으면 작은 적으로 분열 |
| 보스버그 | 높은 체력과 특수 패턴 |

MVP에서는 기본 적, 빠른 적, 보스만 구현합니다.

## 11. 게임 모드

### 11.1 기본 모드

- 정해진 웨이브를 클리어하는 모드
- 초보자에게 적합
- 튜토리얼과 연결하기 좋음

### 11.2 무한 모드

- 실패할 때까지 계속 진행
- 랭킹 경쟁에 적합
- MVP 랭킹용으로 좋음

### 11.3 주간 도전

- 매주 고정 규칙 적용
- 예: 지원형 유닛 등장률 증가, 보스 체력 증가, 합성 비용 감소
- 이벤트 운영에 적합

### 11.4 듀오 협동

- 두 명이 각자의 필드를 갖고 같은 웨이브를 방어
- 실시간 동기화가 필요함
- MVP 이후 확장

### 11.5 서버 레이드

- 서버 전체가 누적 피해를 넣는 비동기 이벤트
- 디스코드 커뮤니티 이벤트와 매우 잘 맞음
- MVP 이후 확장

## 12. 점수 계산

점수는 단순하고 투명해야 합니다.

초기 공식 예시:

```text
최종 점수 = 웨이브 점수 + 처치 점수 + 보스 점수 + 남은 생명 보너스 + 시간 보너스
```

예시 요소:

- 웨이브 도달: 웨이브당 100점
- 처치 수: 적 1마리당 5점
- 보스 처치: 보스당 500점
- 남은 생명: 1당 100점
- 클리어 시간: 빠를수록 보너스

정확한 수치는 밸런스 파일로 분리합니다.

## 13. 실패 조건

추천 실패 조건:

- 적이 끝 지점에 도달하면 생명 감소
- 생명이 0이 되면 게임 종료
- 특정 보스를 제한 시간 안에 잡지 못하면 큰 피해

MVP에서는 생명 20, 적 통과 시 1 감소 같은 단순 구조를 추천합니다.

## 14. 조작 방식

PC:

- 클릭으로 호출
- 클릭으로 유닛 선택
- 합성 가능 유닛 강조
- 버튼으로 합성

모바일:

- 큰 버튼 중심
- 드래그 최소화
- 합성은 선택 후 버튼 또는 자동 추천

## 15. UI 화면 구성

### 15.1 게임 화면

- 상단: 웨이브, 생명, 자원, 점수
- 중앙: 전투 필드
- 하단: 호출 버튼, 강화 버튼, 합성 버튼
- 우측 또는 하단: 현재 보유 유닛 정보

### 15.2 결과 화면

- 최종 점수
- 도달 웨이브
- 랭킹 변화
- 보상
- 다시하기
- 대시보드로 돌아가기
- 디스코드 공유하기

### 15.3 대시보드 화면

- 내 프로필
- 오늘의 랭킹
- 시즌 랭킹
- 최근 플레이 기록
- 이벤트 배너
- 게임 시작 버튼

## 16. MVP 추천 게임 사양

MVP에서 과하게 만들지 않기 위한 추천 사양입니다.

- 유닛 수: 12개
- 등급: 4단계
- 적 타입: 3개
- 보스: 2개
- 웨이브: 30개 또는 무한형 30웨이브 이후 반복 강화
- 유물: 구현 제외 또는 5개만
- 펫/마스코트: 구현 제외
- 모드: 1개
- 랭킹: 개인 최고 점수, 서버 시즌 점수

## 17. 재미 포인트

1. 호출 버튼을 누르는 즉시 랜덤 결과가 나온다.
2. 낮은 등급이 많이 나와도 합성으로 역전 가능하다.
3. 전설급 유닛 등장 시 연출을 준다.
4. 보스 직전 합성 선택이 긴장감을 만든다.
5. 랭킹이 디스코드 커뮤니티 자랑거리로 작동한다.
6. 이벤트 기간에는 특수 룰로 서버 전체가 참여한다.

## 18. 핵심 기획 결론

초기 게임은 복잡한 수집형 RPG가 아니라 `운빨 + 합성 + 웨이브 + 랭킹`에 집중합니다.

MVP의 핵심은 다음 한 문장입니다.

> 디스코드로 로그인한 서버 멤버가 웹에서 랜덤 유닛을 뽑고 합성하며 웨이브를 막고, 점수를 서버 랭킹에 올리는 게임.
