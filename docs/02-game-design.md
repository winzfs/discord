# 02. 게임 기획서

## 1. 장르 정의

장르: **싱글플레이 중심 랜덤 디펜스 웹게임**

핵심 키워드:

- 싱글플레이 메인
- 랜덤 영웅 소환
- 웨이브 방어
- 영웅 합성
- 등급 성장
- 보스 웨이브
- 짧은 판당 플레이
- 운빨과 빌드 선택
- 서버 랭킹
- 디스코드 커뮤니티 연동
- 오버워치 팬게임 감성

## 2. 한 줄 콘셉트

오버워치 영웅과 스킬 감성을 활용해, 디스코드 서버 유저가 혼자 가볍게 즐기는 운빨존많겜 스타일 랜덤 웨이브 디펜스입니다.

## 3. 프로젝트 성격

이 프로젝트는 비상업 팬게임입니다.

- 수익화 없음
- 광고 없음
- 유료 재화 없음
- 유료 시즌패스 없음
- 디스코드 커뮤니티 내부 놀이용
- Blizzard Entertainment와 공식 관련 없음

서비스 화면 또는 README에는 다음 고지 문구를 둘 수 있습니다.

```text
이 프로젝트는 오버워치 팬 커뮤니티를 위한 비상업 팬게임입니다.
오버워치 및 관련 명칭, 캐릭터, 설정은 Blizzard Entertainment의 자산입니다.
본 프로젝트는 Blizzard Entertainment와 공식적으로 관련이 없으며, 후원·제휴·승인을 받은 서비스가 아닙니다.
```

## 4. 플레이 목표

플레이어는 제한된 필드에 랜덤으로 등장하는 영웅을 배치하고, 같은 등급의 영웅을 합성하여 더 높은 등급의 영웅을 만들며, 몰려오는 웨이브를 최대한 오래 막습니다.

MVP 목표:

- 기본 싱글 웨이브 디펜스 1개만 구현
- 모드 선택 화면은 만들지 않음
- 한 판에서 소환, 합성, 웨이브, 보스, 점수 저장을 모두 경험하게 함

성공 기준:

- 목표 웨이브까지 생존하거나
- 생명이 0이 될 때까지 최대한 오래 버티거나
- 최종 점수를 서버 랭킹에 반영

## 5. 핵심 플레이 루프

1. 게임 시작
2. 4x4 필드 생성
3. 초기 자원 지급
4. 랜덤 영웅 소환
5. 적 웨이브 등장
6. 영웅 자동 공격
7. 적 처치 시 자원 획득
8. 자원으로 추가 소환 또는 강화
9. 같은 등급 영웅 3명 합성
10. 상위 등급 영웅 랜덤 획득
11. 보스 웨이브 도전
12. 생명 0 또는 클리어 조건 달성
13. 점수 저장
14. 개인 최고 기록 및 서버 랭킹 반영

## 6. 한 판의 구성

### 6.1 시작 단계

- 플레이어는 빈 필드에서 시작합니다.
- 기본 자원을 받습니다.
- 첫 영웅을 몇 번 무료 또는 저비용으로 소환할 수 있습니다.
- 튜토리얼은 최소화합니다.

필수 안내:

- 소환
- 합성
- 웨이브
- 보스
- 결과 저장

### 6.2 전투 단계

- 적은 정해진 경로를 따라 이동합니다.
- 영웅은 자동으로 공격합니다.
- 플레이어는 전투 중에도 영웅을 소환하거나 합성할 수 있습니다.
- 모바일에서도 누르기 쉬운 큰 버튼 중심으로 구성합니다.

### 6.3 성장 단계

- 적 처치로 자원을 얻습니다.
- 자원으로 영웅을 추가 소환합니다.
- 같은 등급 영웅 3명을 합성합니다.
- 합성 결과는 상위 등급 랜덤 영웅입니다.
- 전투 중 강화는 MVP에서는 단순하게 유지합니다.

### 6.4 보스 단계

- 일정 웨이브마다 보스가 등장합니다.
- 보스는 체력이 높고 제한 시간 압박을 줍니다.
- 보스를 처치하면 큰 점수와 보상을 얻습니다.
- 보스를 놓치면 생명력이 크게 감소하거나 게임이 종료될 수 있습니다.

### 6.5 종료 단계

게임 종료 시 다음 결과를 보여줍니다.

- 도달 웨이브
- 처치 수
- 보스 처치 수
- 최종 점수
- 최고 등급 영웅
- 개인 최고 기록 갱신 여부
- 서버 랭킹 변동
- 유쾌한 결과 평가 문구

## 7. 필드 구조

MVP 기본 필드:

```text
4 x 4 슬롯형 필드
```

이유:

- 운빨존많겜식 슬롯 관리와 잘 맞음
- 모바일에서 보기 쉬움
- 합성 가능한 영웅을 찾기 쉬움
- 초반 구현 난이도가 낮음

규칙:

- 영웅은 빈 슬롯에 자동 배치됩니다.
- 필드가 가득 차면 추가 소환할 수 없습니다.
- 위치 이동은 MVP에서 제외합니다.
- 합성 시 선택된 영웅 3명이 사라지고 상위 영웅 1명이 생성됩니다.

## 8. 자원 구조

### 8.1 전투 자원

전투 중 사용하는 자원입니다.

후보명:

- 코인
- 에너지
- 호출 포인트
- 도파민

사용처:

- 영웅 소환
- 간단한 공격력 강화
- 추후 스킬/궁극기 보조

MVP 추천:

```text
전투 자원명: 코인
```

이유:

- 직관적
- 설명이 쉬움
- 밸런스 조정이 쉬움

### 8.2 계정 자원

MVP에서는 계정 자원 사용처를 만들지 않습니다.

추후 확장:

- 유물 해금
- 프로필 꾸미기
- 시즌 보상
- 칭호/배지

## 9. 영웅 역할

오버워치 역할군을 그대로 이해하기 쉬운 형태로 사용합니다.

| 역할 | 디펜스 내 기능 |
|---|---|
| 탱커 | 적 지연, 실드, 생명 보호, 보스 대응 |
| 딜러 | 단일 피해, 범위 피해, 연속 공격, 보스 딜링 |
| 지원 | 공격속도 증가, 피해 증가, 회복, 자원 보조 |

MVP에서는 별도 제어형/행운형 역할을 만들지 않습니다.

제어 효과와 행운 효과는 특정 영웅의 스킬 또는 태그로 처리합니다.

예시:

```text
role: support
tags: ['heal', 'attack-speed-buff']
```

```text
role: damage
tags: ['area-damage', 'boss-killer']
```

## 10. 영웅 등급

MVP 추천 등급:

| 등급 | 설명 |
|---|---|
| 일반 | 자주 등장, 합성 재료 |
| 희귀 | 초반 전투력 담당 |
| 영웅 | 조합 중심 유닛 |
| 전설 | 판을 바꾸는 핵심 영웅 |

추후 확장 등급:

| 등급 | 설명 |
|---|---|
| 신화 | 매우 낮은 확률, 시즌/이벤트 핵심 |

## 11. 영웅 소환

기본 규칙:

- 자원을 소모해 랜덤 영웅을 소환합니다.
- 빈 슬롯이 있어야 소환할 수 있습니다.
- 소환 비용은 호출 횟수에 따라 증가할 수 있습니다.
- 등장 확률은 등급별 확률표를 따릅니다.

예시 확률:

| 등급 | 확률 |
|---|---:|
| 일반 | 65% |
| 희귀 | 25% |
| 영웅 | 8% |
| 전설 | 2% |

실제 수치는 `packages/game/src/data/balance.ts` 또는 DB 설정으로 분리합니다.

## 12. 합성 규칙

MVP 기본 규칙:

```text
같은 등급 영웅 3명 합성 → 한 단계 높은 등급의 랜덤 영웅 1명
```

예시:

```text
일반 3명 → 희귀 1명
희귀 3명 → 영웅 1명
영웅 3명 → 전설 1명
```

추후 확장 후보:

- 같은 역할 영웅 3명 합성 시 같은 역할 확률 증가
- 특정 영웅 조합 보너스
- 합성 실패 방지 유물
- 합성 비용 환급 효과

## 13. 스킬 구조

각 영웅은 기본 공격과 스킬을 가질 수 있습니다.

스킬 타입:

| 타입 | 설명 |
|---|---|
| 공격 | 단일/범위/연속 피해 |
| 제어 | 둔화, 기절, 밀침, 방어력 감소 |
| 지원 | 공격속도 증가, 피해 증가, 회복, 자원 보조 |
| 궁극기형 | 일정 조건에서 강한 효과 발동 |

MVP 추천:

- 일반: 기본 공격 위주
- 희귀: 간단한 추가 효과
- 영웅: 패시브 또는 확률 효과
- 전설: 고유 스킬 또는 궁극기형 효과

## 14. 궁극기 감성

오버워치 팬게임 느낌을 살리려면 궁극기 감성이 중요합니다.

다만 랜덤 디펜스 장르에 맞게 수동 조작보다 자동/조건부 발동을 우선합니다.

발동 조건 후보:

- 일정 시간마다 발동
- 일정 공격 횟수마다 발동
- 보스 등장 시 발동
- 적이 일정 수 이상 쌓이면 발동
- 합성 직후 발동

예시:

- 전설 딜러가 10초마다 광역 스킬 사용
- 지원 영웅이 보스 등장 시 전체 공격속도 증가
- 탱커 영웅이 서버 코어 피해를 1회 막음

## 15. 적 구성

적은 오버워치 영웅이 아니라 게임 전용 적으로 구성합니다.

추천 적:

| 적 이름 후보 | 특징 |
|---|---|
| 잡버그 | 기본 적 |
| 렉덩어리 | 느리지만 튼튼함 |
| 핑폭탄 | 빠르게 지나감 |
| 트롤봇 | 랜덤 방해 효과 |
| 패치오류체 | 죽으면 작은 버그로 분열 |
| 서버크래셔 | 기본 보스 |
| 새벽큐 악몽 | 이벤트 보스 |

MVP에서는 다음만 구현합니다.

- 잡버그
- 핑폭탄
- 렉덩어리
- 서버크래셔

## 16. 웨이브 구조

MVP 추천:

- 30웨이브 기본 구성
- 5웨이브마다 보스
- 30웨이브 클리어 시 승리 처리 또는 이후 무한 강화 진입

초기에는 30웨이브 클리어 구조가 좋습니다.

이유:

- 한 판 목표가 명확함
- 밸런스 테스트가 쉬움
- 랭킹은 점수와 클리어 시간으로 차별화 가능

추후에는 30웨이브 이후 무한 모드처럼 이어지게 확장할 수 있습니다.

## 17. 점수 계산

점수는 단순하고 투명해야 합니다.

초기 공식 예시:

```text
최종 점수 = 웨이브 점수 + 처치 점수 + 보스 점수 + 남은 생명 보너스 + 시간 보너스
```

예시 요소:

- 웨이브 도달: 웨이브당 100점
- 처치 수: 적 1마리당 5점
- 보스 처치: 보스당 500점
- 남은 생명: 1당 100점
- 클리어 시간: 빠를수록 보너스

정확한 수치는 밸런스 파일로 분리합니다.

## 18. 실패 조건

추천 실패 조건:

- 적이 끝 지점에 도달하면 생명 감소
- 생명이 0이 되면 게임 종료
- 보스를 제한 시간 안에 잡지 못하면 큰 피해

MVP 추천:

```text
생명 20
일반 적 통과 시 -1
보스 통과 시 -5 또는 즉시 실패
```

## 19. 조작 방식

PC:

- 클릭으로 소환
- 클릭으로 영웅 선택
- 합성 가능 영웅 강조
- 버튼으로 합성

모바일:

- 큰 버튼 중심
- 드래그 최소화
- 선택 후 합성 버튼
- 필드 슬롯은 크게 표시

## 20. UI 화면 구성

### 20.1 게임 화면

- 상단: 웨이브, 생명, 코인, 점수
- 중앙: 4x4 영웅 필드
- 전투 경로: 적 이동 라인
- 하단: 소환 버튼, 강화 버튼, 합성 버튼
- 보조 영역: 선택한 영웅 정보

### 20.2 결과 화면

- 최종 점수
- 도달 웨이브
- 처치 수
- 보스 처치 수
- 최고 등급 영웅
- 랭킹 변화
- 다시하기
- 대시보드로 돌아가기
- 디스코드 공유 문구 복사

### 20.3 대시보드 화면

- 내 프로필
- 내 최고 기록
- 서버 랭킹
- 최근 플레이 기록
- 게임 시작 버튼

## 21. 결과 화면 문구 톤

결과 화면은 유쾌하고 공유하고 싶게 만듭니다.

예시:

- 오늘의 팀합: 이상하게 좋음
- 조합 상태: 딜러만 나왔는데 왜 이김?
- 힐러가 안 나왔지만 멘탈로 버팀
- 탱커 없이 20웨이브 돌파. 이게 맞나?
- 전설 하나가 서버를 살렸다
- 억까를 이긴 자
- 이 판은 하이라이트 감이다
- 랜덤이 억까했지만 손가락이 캐리했다

## 22. MVP 추천 게임 사양

- 기본 플레이: 싱글플레이
- 필드: 4x4
- 영웅 수: 12~16명
- 역할: 탱커/딜러/지원
- 등급: 일반/희귀/영웅/전설
- 적 타입: 3종 + 보스 1종
- 웨이브: 30개
- 합성: 같은 등급 3개 → 상위 등급 랜덤 1개
- 랭킹: 개인 최고 점수, 서버 최고 점수
- 2인 협동: MVP 제외, 추후 확장

## 23. 핵심 기획 결론

초기 게임은 복잡한 수집형 RPG나 다중 모드 게임이 아닙니다.

MVP의 핵심은 다음 한 문장입니다.

> 디스코드로 로그인한 서버 멤버가 웹에서 혼자 랜덤 영웅을 뽑고 합성하며 30웨이브를 막고, 점수를 서버 랭킹에 올리는 오버워치 팬게임형 랜덤 디펜스.
