# 25. 히어로 스트라이크 시스템 전면 점검

## 1. 점검 목적

히어로 스트라이크의 로비, 주무기, 보조무기, 성장, 임무, 난이도, 보상, 메타 성장을 하나의 플레이 루프로 다시 점검합니다.

핵심 질문은 다음과 같습니다.

1. 출격 전에 플레이 방식이 명확한가?
2. 주무기를 바꾸면 조작 리듬이 실제로 달라지는가?
3. 성장 선택이 현재 무기의 정체성을 강화하는가?
4. 임무와 보상이 중복 시스템 없이 자연스럽게 연결되는가?
5. 한 판이 반복되기 전에 완주 만족을 주는가?

## 2. 참고한 상용 게임의 공통 원칙

- `Brotato`: 짧은 전투와 상점 선택을 번갈아 배치해 전투 결과가 다음 빌드 결정으로 바로 연결됩니다.
- `20 Minutes Till Dawn`: 캐릭터보다 시작 무기와 직접 조준 리듬이 런 정체성에 큰 영향을 줍니다.
- `Phoenix 2`: 기체마다 공격과 능력의 사용 타이밍이 달라 같은 스테이지도 접근법이 달라집니다.
- `Nova Drift`: 단순 수치 상승보다 기존 공격 규칙을 변형하는 모듈 조합이 빌드 차이를 만듭니다.
- `Deep Rock Galactic: Survivor`: 임무 목표, 클래스 무기, 오버클럭이 서로 연결되어 다음 선택의 방향을 제시합니다.

히어로 스트라이크에는 시스템 수를 늘리는 방식보다 **무기 행동 규칙 → 성장 → 임무 보상**이 한 줄로 이어지는 구조가 더 적합합니다.

## 3. 발견한 핵심 문제

### 3.1 주무기 정체성 평준화

문서상 DRIVE와 FOCUS가 무기마다 달랐지만, 실제 코드에서는 세 무기 모두 양쪽 모드에서 계속 자동 사격했습니다.

특히 아크 레일은 `이동 중 축전 → 정지 후 한 발`이라는 핵심 리듬이 약한 연속탄에 묻혀 있었습니다.

### 3.2 중복된 공격 규칙

- 독립 주무기 런타임과 `player.fireInterval`, `bulletCount`가 동시에 남아 있었습니다.
- 구형 프로토콜이 전역 DRIVE/FOCUS 피해와 사격 간격을 다시 보정했습니다.
- 같은 강화가 프로필과 플레이어 공통 수치를 중복 변경할 가능성이 있었습니다.

### 3.3 보조무기가 주무기 공격력에 종속

미사일, 드론, 측면포가 `player.damage`를 기준으로 계산되어 아크 레일을 선택하면 보조무기까지 과도하게 강해졌습니다.

보조무기 성능이 선택한 주무기 기본 공격력보다 보조무기 자체 레벨에 의해 결정되어야 합니다.

### 3.4 성장 수렴

- 펄스와 레일에는 사실상 전용 진화가 있었지만 산탄에는 없었습니다.
- 진화 조건이 주무기에 귀속되지 않아 다른 무기로도 같은 진화를 열 수 있었습니다.
- 정비소는 수리와 보조무기 강화 위주라 주무기 빌드에 대한 결정을 제공하지 못했습니다.

### 3.5 보상 화면 중복

보스 격파 뒤 구형 프로토콜 3택과 정비소 구매가 연속으로 등장했습니다.

두 화면 모두 런 성장 보상 역할을 수행해 선택 피로를 만들고, 프로토콜의 전역 배율은 새 무기 규칙과 충돌했습니다.

### 3.6 전투 랭크와 임무 보상 중복

임무 성공 보상에 난이도 배율과 전투 랭크 배율이 함께 적용됐습니다.

전투 랭크는 플레이 피드백이어야 하며, 재화 획득은 명확한 임무 성공과 위험도 선택에 연결하는 편이 이해하기 쉽습니다.

### 3.7 런 길이와 반복

10개 구역에서 같은 네 임무가 같은 순서로 반복됐습니다. 구역 전투 시간만 합쳐도 약 10분이며 보스와 정비를 포함하면 목표한 10~12분을 크게 넘길 가능성이 높았습니다.

## 4. 적용한 설계

### 4.1 주무기 행동 계약

| 주무기 | DRIVE | FOCUS | 핵심 자원 |
|---|---|---|---|
| 펄스 리피터 | 3점사, 높은 냉각 | 5점사, 낮은 냉각 | 열 |
| 브리처 | 넓은 2펠릿 견제 | 좁은 6펠릿 집중 | 5발 탄창 |
| 아크 레일 | 축전만 수행 | 진입 시 1회 관통포 | 축전량 |

### 4.2 무기별 성장 변환

- `사이클 튜닝`: 점사·냉각 / 펌프·재장전 / 축전 속도
- `발사 구조 확장`: 버스트 탄 수 / 펠릿 수 / 보조 광선
- 정비소 `주무기 튜닝`: 두 성장축 중 낮은 쪽을 보완

### 4.3 전용 진화

- 펄스 리피터: `펄스 스톰`
- 브리처: `브리치 노바`
- 아크 레일: `썬더 라인`

주무기 전용 진화는 해당 주무기를 장착했을 때만 해금합니다.

### 4.4 보조무기 독립화

보조무기는 고정 기준 피해에서 시작하고 `고출력 코어`의 일부 효과만 별도 비율로 받습니다.

주무기 선택은 보조무기의 사용 방식과 표적 공유에는 영향을 주지만, 주무기 기본 공격력으로 보조 DPS가 폭증하지 않습니다.

### 4.5 보스 후 선택 통합

구형 프로토콜 선택 화면을 제거하고 정비소 하나로 통합합니다.

정비소 선택:

1. 긴급 수리
2. 주무기 튜닝
3. 선택 보조무기 튜닝

구역마다 한 번만 구매할 수 있어 회복과 화력 사이의 비용 선택을 유지합니다.

### 4.6 전투 랭크 역할 변경

- 임무 보상 배율에서 제외
- 피격, 그레이즈, BREAK, 임무 성공을 중심으로 계산
- 일반 처치만 반복해도 자동으로 S가 되지 않도록 조정
- HUD와 결과 보고서의 숙련도 피드백으로 사용

### 4.7 5구역 작전

전체 런을 다음 5개 핵심 구역으로 압축합니다.

1. KING'S ROW
2. NULL SECTOR FACTORY
3. JUNKERTOWN SKYWAY
4. RIALTO NIGHT
5. GIBRALTAR ORBIT

각 구역은 세 일반 임무의 순서를 교차하고 마지막에 엘리트와 보스를 배치합니다.

예상 플레이 시간은 전투, 보스, 정비 선택을 합쳐 약 10분입니다.

## 5. 밸런스 기준

### 주무기

- 펄스: 가장 안정적인 기준 화력
- 브리처: 원거리 손실을 감수하고 근거리에서 기준 화력을 초과
- 레일: 단일 대상 DPS보다 관통과 BREAK, 완충 한 발의 가치가 높음

### 보조무기

- 주무기와 무관하게 비슷한 기본 기여도를 가짐
- 초기에는 전체 화력의 약 20~30%를 목표
- 진화 이후에는 빌드에 따라 35~45%까지 허용

### 난이도

- 신병: 체력과 탄속을 낮추고 출현 간격을 늘림
- 요원: 기준값
- 전설: 체력 증가는 제한하고 탄속과 출현 밀도를 높임

## 6. 출시 전 측정 항목

다음 수치는 실제 플레이 로그로 확인해야 합니다.

- 주무기별 구역 클리어 시간
- 주무기별 보스 BREAK 횟수와 보스 처치 시간
- DRIVE/FOCUS 체류 비율
- 산탄 평균 교전 거리와 펠릿 적중률
- 레일 평균 축전량과 완충 발사 비율
- 보조무기별 전체 피해 기여도
- 임무 종류별 성공률
- 정비소 선택률과 미사용 SALVAGE
- 진화 해금 구역과 해금률
- 난이도별 완주율, 피격 수, 평균 플레이 시간

## 7. 다음 개선 우선순위

1. 개발용 전투 통계 패널 또는 런 결과 텔레메트리 추가
2. 주무기별 첫 출격 튜토리얼을 1문장 오버레이로 제공
3. 임무 종류별 적 편성과 탄막 패턴을 더 강하게 연결
4. 블루프린트 해금 시 무기 시험 사격 화면 추가
5. 결과 화면에서 주무기 피해와 보조무기 피해를 분리 표시
