# 13. 업로드 참고 문서 기반 시스템 추가 계획

## 1. 목적

사용자가 업로드한 참고 문서 5종과 이후 추가된 상세 기획 문서를 기준으로 현재 저장소에 필요한 시스템을 정리하고, 구현 완료/미구현 상태를 관리합니다.

참고 문서는 다음 범위를 다룹니다.

- 전체 게임 개요
- 게임 모드/난이도
- 영웅/신화/불멸/보물
- 유물
- 펫
- 같은 캐릭터 3중첩 보드
- 드래그 이동/자리 교환
- 합성/판매 메뉴

이 문서는 원본 내용을 그대로 복제하기 위한 문서가 아니라, 모바일 랜덤 디펜스 장르의 핵심 구조를 우리 프로젝트에 맞게 구현하기 위한 기능 백로그입니다.

## 4. 전투 중 핵심 시스템

### 4.1 이중 전투 재화

현재 전투 중 재화는 다음과 같이 구성되어 있습니다.

| 재화 | 현재 상태 | 용도 |
|---|---|---|
| 코인 | 구현 완료 | 일반 소환, 공격력 강화, 판매 보상 |
| 행운석 | 구현 완료 | 도박 소환, 이후 오버드라이브 조합/고등급 강화 |

추후 추가 예정 재화:

| 재화 | 용도 | 우선순위 |
|---|---|---|
| 초대장 | 전투 밖 영웅 모집 | 이후 |
| 유물 열쇠 | 유물 뽑기 | 이후 |
| 배터리 | 펫 뽑기 | 이후 |
| 펫 푸드 | 펫 강화 | 이후 |
| 신화석 | 신화/오버드라이브 성장 | 이후 |
| 보물 재화 | 고급 성장/장비 | 이후 |

### 4.2 일반 소환

현재 일반 소환은 코인을 사용합니다.

구현 규칙:

```text
소환 클릭
→ 코인 차감
→ 등급 확률표로 등급 결정
→ 해당 등급 영웅 풀에서 랜덤 선택
→ 같은 캐릭터 칸이 3마리 미만이면 우선 중첩
→ 없으면 빈 칸 배치
→ 없으면 실패
```

### 4.3 도박 소환

현재 도박은 `영웅 도박` 1종이 `/play` 버튼에 연결되어 있습니다.

데이터상 준비된 도박 종류:

| 도박 종류 | 비용 | 성공 보상 | 실패 보정 |
|---|---:|---|---|
| 희귀 도박 | 행운석 1 | 희귀 유닛 | 일반 유닛 |
| 영웅 도박 | 행운석 2 | 영웅 유닛 | 희귀 유닛 |
| 전설 도박 | 행운석 4 | 전설 유닛 | 영웅 유닛 |

다음 작업:

```text
도박 버튼 클릭
→ 희귀/영웅/전설 선택 팝업
→ 비용/성공률 표시
→ 선택한 도박 실행
```

### 4.4 공격력 강화

현재 공격력 강화는 전체 전투력 배율을 올리는 1종 강화입니다.

```text
공격력 강화 클릭
→ 코인 차감
→ powerUpgradeLevel 증가
→ calculateBoardPower()에서 강화 배율 반영
```

다음 작업:

```text
공격력 강화 팝업
→ 일반/희귀/영웅/전설/오버드라이브 강화 분리
→ 등급별 비용과 재화 분리
```

### 4.5 같은 캐릭터 3중첩 합성

현재 3중첩 합성은 유닛 메뉴에서 실행합니다.

```text
같은 캐릭터 3마리 중첩
→ 유닛 짧게 탭
→ [합성] [판매] 메뉴 표시
→ 합성 클릭
→ 3마리 소모
→ 상위 등급 랜덤 유닛 1마리 생성
```

현재 결과는 `같은 캐릭터 계열 승급`이 아니라 `상위 등급 풀에서 랜덤 1마리`입니다.

추후 개선 후보:

```text
같은 캐릭터 계열 승급
역할 유지 랜덤 승급
합성 대성공/실패
3중첩 유지 보너스와 합성 선택의 전략성 강화
```

### 4.6 판매

현재 판매는 유닛 메뉴에서 실행합니다.

```text
유닛 짧게 탭
→ [판매]
→ 선택 칸의 맨 위 유닛 1마리 판매
→ 등급별 코인 지급
```

현재 판매 보상:

| 등급 | 보상 |
|---|---:|
| 일반 | 8 코인 |
| 희귀 | 18 코인 |
| 영웅 | 45 코인 |
| 전설 | 110 코인 |

## 6. 오버드라이브 조합 시스템

현재 신화/오버드라이브는 데이터 뼈대만 있습니다.

참고 문서 기준 방향:

- 오버드라이브 유닛은 일반 소환으로 직접 나오지 않습니다.
- 특정 재료 유닛 조합으로 제작합니다.
- 재료는 일반/희귀/영웅/전설 등급 유닛입니다.
- 전설 재료 기반 오버드라이브는 강하지만 제작 난도가 높습니다.
- 영웅 등급 재료 기반 오버드라이브는 초중반 핵심입니다.

우리 프로젝트 재해석:

| 타입 | 재료 구조 | 역할 |
|---|---|---|
| 초반형 | 영웅 2~3종 + 희귀 1종 | 초반 라인 정리 |
| 보스형 | 전설 1종 + 영웅 2종 | 보스 딜 |
| 광역형 | 전설 1종 + 영웅/희귀 조합 | 웨이브 정리 |
| 서포트형 | 영웅 서포터 + 탱커 재료 | 공속/마나/감속 |
| 경제형 | 특정 일반/희귀/영웅 조합 | 코인/행운석 보조 |

다음 구현 방식:

```text
/play의 신화 버튼 클릭
→ 오버드라이브 조합 목록 팝업
→ 재료가 충족된 항목은 밝게 표시
→ 제작 시 재료 유닛 제거
→ 오버드라이브 유닛 생성
```

## 7. 전투 연출/웨이브 시스템

현재 웨이브는 전투력 기반 결과 판정입니다.

현재 흐름:

```text
웨이브 버튼 클릭
→ 몬스터 이동 연출 일부 표시
→ completeCurrentWave(startWave(state)) 실행
→ 전투력/위협도 판정
→ 보상/누수/패배 표시
```

다음 구현 목표:

```text
웨이브 버튼 클릭
→ 적들이 길을 따라 이동
→ 보드 유닛들이 자동 공격
→ 투사체/타격 이펙트 표시
→ 일정 시간 후 기존 전투력 판정 적용
→ 웨이브 결과 표시
```

장기적으로는 웨이브 단위 판정에서 실시간 전투 시뮬레이션으로 확장합니다.

## 8. 영웅 계정 성장 시스템

현재 미구현입니다.

필요 구조:

```text
일반~전설 영웅은 조각으로 레벨업
오버드라이브 영웅은 별도 재화 또는 조각으로 레벨업
특정 레벨에서 해금 효과 부여
```

레벨 해금 구조 예시:

| 레벨 | 효과 |
|---:|---|
| 3 | 공격력 증가 |
| 6 | 고유 효과 1 해금 |
| 9 | 공격 속도 증가 |
| 12 | 고유 효과 2 해금 |
| 15 | 공격력/공격속도 추가 증가 |

MVP 이후 `/heroes` 도감 화면에 연결합니다.

## 9. 공격 유형/빌드 분화

현재 `attackType`은 존재하지만 전투 판정에서 충분히 분화되지 않았습니다.

추가해야 할 빌드 축:

| 축 | 설명 |
|---|---|
| 물리 피해 | 방어력/방깎과 연계 |
| 마법 피해 | 방어력 무시, 마나/궁극기와 연계 |
| 고정 피해 | 일부 방어/감소 효과 무시 |
| 기절 | 라인 밀림 방지 |
| 둔화 | 기절 효율 상승, 이동 제어 |
| 방어력 감소 | 물리 딜러 보조 |
| 마나 회복 | 궁극기형 마법 딜러 보조 |
| 공격 속도 | 평타/기절/발동형 효과 강화 |
| 경제 | 코인/행운석 수급 증가 |

|---|---|
| 공격형 | 전체 공격력 증가 |
| 물리형 | 물리 피해 증가, 방어력 감소 보조 |
| 마법형 | 마법 피해 증가, 마나 회복 보조 |
| 행운형 | 행운석 획득 확률 증가 |
| 경제형 | 코인 보상 증가 |
| 방어형 | 코어 HP 증가, 누수 피해 감소 |
| 제어형 | 기절/둔화 시간 증가 |

펫 슬롯 구조:

| 종합 펫 레벨 | 장착 슬롯 |
|---:|---:|
| 기본 | 1 |
| 일정 레벨 달성 | 2 |
| 높은 종합 레벨 달성 | 3 |

## 12. 게임 모드/난이도

현재 데이터 뼈대만 있습니다.

구현 파일:

```text
packages/game/src/data/gameModes.ts
```

장기 모드 구조:

| 모드 | 설명 | 우선순위 |
|---|---|---|
| 기본 싱글 | 현재 MVP 모드 | 현재 |
| 협동형 | 2인 보드 또는 공동 방어 | 이후 |

디스코드 커뮤니티 프로젝트이므로 장기적으로는 `서버전`, `길드전`, `시즌 랭킹`이 잘 맞습니다.
