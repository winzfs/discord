# 13. 업로드 참고 문서 기반 시스템 추가 계획

## 1. 목적

사용자가 업로드한 참고 문서 5종을 기준으로 현재 저장소에 부족한 시스템을 정리하고, 우리 프로젝트에 맞게 재해석한 추가 설계 목록을 정의합니다.

참고 문서는 다음 범위를 다룹니다.

- 전체 게임 개요
- 게임 모드/난이도
- 영웅/신화/불멸/보물
- 유물
- 펫

이 문서는 원본 내용을 그대로 복제하기 위한 문서가 아니라, 모바일 랜덤 디펜스 장르의 핵심 구조를 우리 프로젝트에 맞게 구현하기 위한 기능 백로그입니다.

## 2. 현재 저장소에 이미 있는 것

현재 저장소에는 다음 요소가 있습니다.

- 싱글플레이 랜덤 디펜스 방향
- `/play` PixiJS 전투 화면
- 20칸 보드 구조
- 코인 기반 일반 소환
- 등급 기반 영웅 데이터
- 웨이브 데이터
- 전투력 기반 웨이브 판정
- 코어 HP
- 하단 신화/소환/도박/강화 버튼 틀
- 유닛 수 표시
- 전투력 표시

## 3. 업로드 문서 기준으로 부족한 핵심 시스템

### 3.1 이중 전투 재화

현재 저장소에는 코인만 있습니다.

추가해야 할 재화:

| 재화 | 용도 | 구현 우선순위 |
|---|---|---|
| 코인 | 일반 소환, 일반~영웅 강화, 일부 상점 | MVP |
| 행운석 | 도박 소환, 전설~신화 강화, 신화 조합 보조 | MVP |
| 초대장 | 전투 밖 영웅 모집 | 이후 |
| 유물 열쇠 | 유물 뽑기 | 이후 |
| 배터리 | 펫 뽑기 | 이후 |
| 펫 푸드 | 펫 강화 | 이후 |
| 신화석 | 신화 영웅 성장 | 이후 |
| 보물 재화 | 신화 영웅 보물 성장 | 이후 |

즉시 반영할 것은 `luckStones`입니다.

### 3.2 전투 중 미션 시스템

참고 문서에는 난이도별로 전투 중 미션이 존재합니다.

우리 프로젝트에 필요한 미션 예시:

| 미션 | 보상 | 목적 |
|---|---:|---|
| 모든 일반 등급 발견 | 코인 | 초반 소환 유도 |
| 모든 희귀 등급 발견 | 코인 | 수집 유도 |
| 영웅 등급 유닛 등장 | 행운석 | 고등급 등장 보상 |
| 전설 등급 유닛 등장 | 행운석 | 희귀한 성공 경험 |
| 코인 일정량 수집 | 코인 | 경제 빌드 보상 |
| 행운석 일정량 수집 | 코인 | 행운석 시스템 인지 |
| 도박 소환 시도 | 행운석 | 도박 버튼 사용 유도 |
| 도박 소환 실패 | 코인 | 실패 보정 |
| 전설 도박 성공 | 코인 | 고점 보상 |
| 강화 일정 횟수 | 코인 | 강화 버튼 사용 유도 |
| 몬스터 일정 수 처치 | 코인 | 전투 진행 보상 |

MVP에서는 미션 5~8개만 구현하고, 이후 난이도별 미션 테이블로 확장합니다.

### 3.3 도박 소환 시스템

현재 하단에 `도박` 버튼만 있습니다.

추가해야 할 구조:

| 도박 종류 | 비용 | 성공 보상 | 실패 보상 |
|---|---:|---|---|
| 희귀 도박 | 낮은 행운석 | 희귀 유닛 | 없음 또는 소량 코인 |
| 영웅 도박 | 중간 행운석 | 영웅 유닛 | 실패 보정 |
| 전설 도박 | 높은 행운석 | 전설 유닛 | 낮은 등급 보정 가능 |

설계 방향:

- 도박은 일반 소환과 별도 확률을 사용합니다.
- 실패가 너무 허무하지 않도록 유물/미션/보정 시스템을 붙입니다.
- 도박은 신화 재료 수급의 핵심 버튼으로 사용합니다.

### 3.4 등급별 강화 시스템

참고 문서에는 전투 중 등급별 공격력 강화가 있습니다.

우리 프로젝트 기본 구조:

| 강화 대상 | 비용 | 효과 |
|---|---:|---|
| 일반/희귀 강화 | 코인 | 해당 등급 공격력 증가 |
| 영웅 강화 | 코인 | 해당 등급 공격력 증가 |
| 전설 강화 | 행운석 | 해당 등급 공격력 증가 |
| 신화 강화 | 행운석 | 신화 공격력 증가 |

MVP에서는 하나의 `공격력 강화` 버튼으로 시작하고, 이후 등급별 강화 팝업으로 분리합니다.

### 3.5 신화 조합 시스템

현재 저장소에는 단순 등급 합성만 있습니다.

참고 문서 기준으로 중요한 방향:

- 신화 유닛은 일반 소환으로 직접 나오지 않음
- 특정 재료 유닛 조합으로 제작
- 재료는 일반/희귀/영웅/전설 등급 유닛
- 전설 재료 기반 신화는 강하지만 제작 난도가 높음
- 영웅 등급 재료 기반 신화는 초중반 핵심

우리 프로젝트 재해석:

| 신화 타입 | 재료 구조 | 역할 |
|---|---|---|
| 초반형 신화 | 영웅 2~3종 + 희귀 1종 | 초반 라인 정리 |
| 보스형 신화 | 전설 1종 + 영웅 2종 | 보스 딜 |
| 광역형 신화 | 전설 1종 + 영웅/희귀 조합 | 웨이브 정리 |
| 서포트형 신화 | 영웅 서포터 + 탱커 재료 | 공속/마나/감속 |
| 경제형 신화 | 특정 일반/희귀/영웅 조합 | 코인/행운석 보조 |

구현 방식:

- `/play`의 `신화` 버튼을 누르면 제작 가능한 신화 목록 팝업
- 재료가 충족된 항목은 밝게 표시
- 제작 시 재료 유닛 제거 후 신화 유닛 생성
- 신화 유닛은 별도 등급 또는 `legendary` 위 단계로 확장 필요

### 3.6 영웅 계정 성장 시스템

참고 문서에는 영웅별 계정 레벨이 존재합니다.

우리 프로젝트에 필요한 구조:

- 일반~전설 영웅은 조각으로 레벨업
- 신화 영웅은 별도 재화 또는 조각으로 레벨업
- 특정 레벨에서 해금 효과 부여

레벨 해금 구조 예시:

| 레벨 | 효과 |
|---:|---|
| 3 | 공격력 증가 |
| 6 | 고유 효과 1 해금 |
| 9 | 공격 속도 증가 |
| 12 | 고유 효과 2 해금 |
| 15 | 공격력/공격속도 추가 증가 |

MVP 이후 `/heroes` 도감 화면에 연결합니다.

### 3.7 공격 유형/빌드 분화

현재 저장소에는 `attackType`이 있지만 전투 판정에서 충분히 분화되지 않았습니다.

추가해야 할 빌드 축:

| 축 | 설명 |
|---|---|
| 물리 피해 | 방어력/방깎과 연계 |
| 마법 피해 | 방어력 무시, 마나/궁극기와 연계 |
| 고정 피해 | 일부 방어/감소 효과 무시 |
| 기절 | 라인 밀림 방지 |
| 둔화 | 기절 효율 상승, 이동 제어 |
| 방어력 감소 | 물리 딜러 보조 |
| 마나 회복 | 궁극기형 마법 딜러 보조 |
| 공격 속도 | 평타/기절/발동형 효과 강화 |
| 경제 | 코인/행운석 수급 증가 |

### 3.8 유물 시스템

현재 저장소에는 유물 시스템이 없습니다.

참고 문서 기반 핵심 구조:

- 유물은 전투 밖에서 수집하는 패시브 시스템
- 유물은 열쇠로 무작위 획득
- 중복 획득으로 레벨업
- 특정 유물은 게임 운영을 크게 바꿈

우리 프로젝트 유물 카테고리:

| 카테고리 | 효과 예시 |
|---|---|
| 공격 | 전체 공격력, 물리 피해, 마법 피해 증가 |
| 경제 | 웨이브마다 코인 획득, 보유 코인 비례 공격력 |
| 행운 | 행운석 획득 증가, 도박 성공률 증가 |
| 소환 | 희귀/영웅/전설 등장률 증가 |
| 방어 | 코어 HP 증가, 누수 피해 감소 |
| 제어 | 기절/둔화 효율 증가 |
| 보정 | 도박 실패 시 환급/하위 등급 지급 |

MVP 이후 `artifactCatalog` 데이터로 구현합니다.

### 3.9 펫 시스템

현재 저장소에는 펫 시스템이 없습니다.

참고 문서 기반 핵심 구조:

- 계정 레벨 달성 후 해금
- 펫은 별도 재화로 조각 뽑기
- 잠금 해제 후 장착 가능
- 펫은 공격력 증가와 부가 효과를 제공
- 펫 레벨과 등급에 따라 효과 증가
- 전체 펫 종합 레벨에 따라 장착 슬롯 증가

우리 프로젝트 펫 카테고리:

| 카테고리 | 효과 예시 |
|---|---|
| 공격형 | 전체 공격력 증가 |
| 물리형 | 물리 피해 증가, 방어력 감소 보조 |
| 마법형 | 마법 피해 증가, 마나 회복 보조 |
| 행운형 | 행운석 획득 확률 증가 |
| 경제형 | 코인 보상 증가 |
| 방어형 | 코어 HP 증가, 누수 피해 감소 |
| 제어형 | 기절/둔화 시간 증가 |

펫 슬롯 구조:

| 종합 펫 레벨 | 장착 슬롯 |
|---:|---:|
| 기본 | 1 |
| 일정 레벨 달성 | 2 |
| 높은 종합 레벨 달성 | 3 |

### 3.10 게임 모드/난이도

현재 저장소에는 단일 싱글플레이 모드만 있습니다.

추가할 모드 구조:

| 모드 | 설명 | 우선순위 |
|---|---|---|
| 기본 싱글 | 현재 MVP 모드 | 현재 |
| 협동형 | 2인 보드 또는 공동 방어 | 이후 |
| 대전형 | 상대 필드와 경쟁 | 이후 |
| 도전형 | 특수 규칙, 제한 조건 | 이후 |
| 무한형 | 끝없는 웨이브 랭킹 | 이후 |
| 오늘의 던전 | 일일 변형 규칙 | 이후 |
| 길드/서버전 | 디스코드 서버 단위 경쟁 | 장기 |

디스코드 커뮤니티 프로젝트이므로 장기적으로는 `서버전`, `길드전`, `시즌 랭킹`이 가장 잘 맞습니다.

## 5. 개발 우선순위

### 5.1 바로 구현

1. `luckStones` 상태 추가
2. 웨이브 보상에 행운석 보상 일부 추가
3. 도박 소환 시스템 추가
4. 공격력 강화 시스템 추가
5. 하단 `도박`, `강화` 버튼 실제 작동
6. 신화 조합 데이터 구조 추가

### 5.2 다음 구현

1. 전투 중 미션 시스템
2. 신화 조합 팝업
3. 등급별 강화 팝업
4. 유물 카탈로그 데이터
5. 펫 카탈로그 데이터

### 5.3 이후 구현

1. 영웅 도감/레벨업
2. 유물 뽑기/레벨업
3. 펫 뽑기/장착
4. 계정 저장/랭킹/시즌
5. 디스코드 OAuth 연동
6. 서버전/길드전

## 6. 주의사항

- 원본 문서의 캐릭터명, 수치, 고유 스킬을 그대로 복제하지 않습니다.
- 시스템 구조와 장르적 흐름만 참고합니다.
- 오버워치 커뮤니티 팬게임 성격에 맞게 영웅, 유물, 펫 이름과 효과는 별도로 설계합니다.
- 우선은 싱글플레이 MVP의 재미를 완성하고, 그다음 메타 성장과 디스코드 연동을 붙입니다.
