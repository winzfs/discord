# 04. 디스코드 연동 대시보드 기획

## 1. 목적

대시보드는 단순한 게임 실행 화면이 아니라, 디스코드 커뮤니티 서버의 활동 허브 역할을 합니다.

주요 목적:

- 디스코드 계정으로 로그인
- 서버 멤버 인증
- 개인 게임 기록 확인
- 서버 랭킹 확인
- 시즌/이벤트 안내
- 관리자용 운영 도구 제공
- 추후 디스코드 봇과 연동

## 2. 주요 사용자 흐름

### 2.1 일반 유저 흐름

1. 디스코드 서버 공지 또는 봇 명령어로 웹 링크 접속
2. Discord OAuth 로그인
3. 서버 멤버 여부 확인
4. 대시보드 홈 진입
5. 내 프로필, 랭킹, 이벤트 확인
6. 게임 시작
7. 게임 종료 후 결과 저장
8. 랭킹 반영
9. 결과 공유 또는 다시 플레이

### 2.2 관리자 흐름

1. 관리자 계정으로 로그인
2. Discord 서버 권한 확인
3. 관리자 대시보드 진입
4. 시즌 상태 확인
5. 랭킹 확인
6. 이벤트 배너/공지 관리
7. 문제 유저 기록 확인
8. 필요 시 기록 숨김 또는 초기화

## 3. 로그인/인증

### 3.1 Discord OAuth2

로그인은 Discord OAuth2를 사용합니다.

필요 Scope 후보:

- `identify`: 유저 기본 정보 확인
- `guilds`: 유저가 속한 서버 확인
- `guilds.members.read`: 특정 서버 멤버 상세 확인이 필요할 경우 사용 검토

MVP에서는 다음을 목표로 합니다.

- 유저 Discord ID 저장
- 유저 닉네임/아바타 저장
- 지정 서버 멤버인지 확인
- 관리자 권한 여부 확인

### 3.2 세션 처리

Cloudflare 환경에서 세션은 다음 중 하나를 사용합니다.

1. 암호화된 쿠키 기반 세션
2. Cloudflare KV 기반 세션 저장
3. JWT + 짧은 만료 시간

MVP 추천:

- 암호화 쿠키 또는 JWT
- 민감 정보는 쿠키에 직접 저장하지 않음
- Discord access token은 가능하면 짧게 사용하고 저장 최소화

### 3.3 저장할 유저 정보

```ts
type User = {
  id: string;
  discordId: string;
  username: string;
  globalName?: string;
  avatarUrl?: string;
  joinedAt: string;
  lastLoginAt: string;
  isGuildMember: boolean;
  isAdmin: boolean;
};
```

## 4. 서버 멤버 인증

### 4.1 인증 기준

서비스는 특정 디스코드 서버 멤버만 접근 가능하게 할 수 있습니다.

검증 방식:

- 로그인한 유저의 guild 목록에 목표 서버 ID가 있는지 확인
- 또는 Discord API로 guild member endpoint 확인

### 4.2 비멤버 처리

비멤버 접근 시:

- 서비스 이용 제한
- 서버 초대 링크 표시
- 가입 후 다시 로그인 안내

주의:

- 서버 초대 링크는 환경 변수 또는 관리자 설정으로 관리
- 공개 저장소에 민감한 초대 링크를 하드코딩하지 않음

## 5. 대시보드 화면 구성

### 5.1 홈

표시 요소:

- 유저 프로필 카드
- 오늘의 최고 기록
- 시즌 랭킹 요약
- 게임 시작 버튼
- 진행 중 이벤트
- 최근 공지

### 5.2 내 프로필

표시 요소:

- Discord 닉네임/아바타
- 총 플레이 횟수
- 최고 웨이브
- 최고 점수
- 시즌 최고 점수
- 보유 칭호
- 보유 배지
- 최근 플레이 기록

### 5.3 랭킹

랭킹 종류:

- 오늘의 랭킹
- 주간 랭킹
- 시즌 랭킹
- 전체 최고 기록

MVP에서는 시즌 랭킹과 전체 최고 기록만 구현합니다.

랭킹 항목:

- 순위
- 닉네임
- 아바타
- 최고 점수
- 도달 웨이브
- 기록 일시

### 5.4 게임 시작 페이지

- 모드 선택
- 현재 시즌 룰 표시
- 내 최고 기록 표시
- 시작 버튼
- 조작법 간단 안내

MVP에서는 모드 선택 없이 바로 시작해도 됩니다.

### 5.5 결과 페이지

- 최종 점수
- 도달 웨이브
- 처치 수
- 보스 처치 수
- 획득 보상
- 랭킹 변화
- 다시하기
- 결과 공유

### 5.6 관리자 페이지

MVP 관리자 기능:

- 전체 유저 수
- 전체 플레이 수
- 최근 플레이 기록
- 시즌 랭킹 확인
- 특정 유저 검색

1차 확장 관리자 기능:

- 시즌 생성/종료
- 이벤트 배너 설정
- 랭킹 기록 숨김
- 보상 지급 기록 확인
- 서버 공지 문구 관리

## 6. 디스코드 봇 연동 후보

웹만으로도 MVP는 가능하지만, 디스코드 봇을 붙이면 커뮤니티성이 강해집니다.

명령어 후보:

| 명령어 | 설명 |
|---|---|
| `/게임` | 웹게임 링크 표시 |
| `/랭킹` | 현재 시즌 TOP 10 표시 |
| `/내기록` | 자신의 최고 기록 표시 |
| `/이벤트` | 진행 중 이벤트 표시 |
| `/보상` | 시즌 보상 안내 |

봇은 MVP 이후 추가해도 됩니다.

## 7. 결과 공유 기능

게임 종료 후 디스코드 공유용 문구를 만들 수 있습니다.

예시:

```text
🎲 도파민 디펜스 결과
웨이브: 28
점수: 42,500
보스 처치: 4
서버 랭킹: 7위
```

구현 방식:

- 클립보드 복사
- 디스코드 채널 웹훅 전송
- 봇 명령어와 연동

MVP에서는 클립보드 복사만 추천합니다.

## 10. 보안 고려사항

1. Discord client secret은 절대 저장소에 커밋하지 않는다.
2. 관리자 API는 반드시 권한 체크를 거친다.
3. 게임 결과 저장 시 클라이언트 결과를 그대로 신뢰하지 않는다.
4. 너무 빠른 반복 기록 제출은 rate limit 처리한다.
5. 비정상 점수는 관리자 검토 대상으로 표시한다.
6. 유저 Discord ID는 공개 화면에서 과도하게 노출하지 않는다.

## 11. 치팅 방지 방향

웹게임은 클라이언트 변조 가능성이 있습니다.

MVP에서는 완벽한 치팅 방지보다 다음을 우선합니다.

- 플레이 시간과 점수의 비정상 조합 감지
- 웨이브 대비 비정상 처치 수 감지
- 너무 잦은 결과 제출 제한
- 결과 payload 저장 후 관리자 검토 가능
- 랭킹 상위권 기록은 suspicious flag 표시

추후 개선:

- 서버 시드 기반 전투 재현
- 핵심 전투 로직 일부 서버 검증
- 리플레이 데이터 저장

## 12. 대시보드 MVP 범위

MVP 구현:

- Discord 로그인
- 서버 멤버 확인
- 홈 화면
- 게임 시작 버튼
- 내 최고 기록
- 시즌 랭킹
- 결과 저장
- 관리자용 최근 기록 목록

MVP 제외:

- 봇 명령어
- 웹훅 자동 공유
- 보상 지급 자동화
- 관리자 시즌 생성 UI
- 마스코트/유물 인벤토리

## 13. 확장 방향

1. 디스코드 봇 추가
2. 시즌 보상과 디스코드 역할 연동
3. 주간 이벤트 자동 공지
4. 서버 전체 레이드 진행도 표시
5. 커뮤니티 미션 추가
6. 관리자 대시보드 고도화
