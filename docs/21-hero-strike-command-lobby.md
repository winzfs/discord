# 21. 히어로 스트라이크 커맨드 로비

## 1. 목표

출격 준비 화면을 옵션 목록이 아니라 `작전 지휘실`로 보이게 만듭니다.

플레이어는 로비에서 다음 질문에 즉시 답을 얻어야 합니다.

1. 나는 어떤 영웅과 장비로 출격하는가?
2. 첫 작전은 어디이며 어떤 위험이 있는가?
3. 선택한 장비는 플레이 방식을 어떻게 바꾸는가?
4. 이번 위험도에서 보상은 얼마나 증가하는가?
5. 다음 Blueprint 해금까지 얼마나 남았는가?

## 2. 화면 구조

```text
COMMAND STATUS
영웅 쇼케이스 | 현재 작전 브리핑
장비 카테고리 탭
선택 카테고리의 장비 3종
현재 장착 슬롯 요약
Blueprint 진행도
DEPLOY
```

기존처럼 네 카테고리의 카드 12장을 한 화면에 모두 노출하지 않습니다. 카테고리 탭을 눌러 한 번에 세 장만 비교하고, 남는 공간을 작전 정보와 계정 진행에 사용합니다.

## 3. 정보 위계

### 3.1 상단 상태 바

- `COMMAND DECK // READY`
- Blueprint Rank
- Research Data
- 최고 점수

### 3.2 영웅 쇼케이스

- 트레이서 스프라이트 확대 표시
- 역할: STRIKE PILOT
- DRIVE / FOCUS 조작 특성
- 블링크 2회, 펄스 궁극기

### 3.3 작전 브리핑

첫 구역인 KING'S ROW 기준으로 표시합니다.

- 지역명과 부제
- 일반 인카운터 수
- 보스 이름
- 예상 작전 시간
- 선택 위험도의 적 압박과 보상 배율

### 3.4 장비 탭

- PRIMARY
- SUPPORT
- TACTICAL
- RISK

현재 탭은 강한 강조색과 하단 인디케이터를 사용합니다.

### 3.5 장비 카드

각 카드는 다음을 표시합니다.

- 아이콘과 이름
- 역할 설명
- 핵심 수치 또는 효과
- 잠금 Rank
- 장착 상태

선택된 장비는 카드 전체가 발광하고 `EQUIPPED` 상태가 됩니다.

### 3.6 현재 장착 슬롯

선택 장비 네 개를 작은 슬롯으로 다시 보여줘 출격 직전 구성을 한눈에 확인할 수 있게 합니다.

### 3.7 Blueprint 진행

- 현재 Rank
- 다음 해금 장비
- Research Data 진행 바
- 현재 해금 수 / 전체 Blueprint 수

### 3.8 DEPLOY

화면 하단 전체 폭을 사용합니다.

- 작전명
- 위험도
- 예상 보상 배율
- `HOLD TO DEPLOY`처럼 보이되 현재 입력 체계에서는 한 번 터치로 출격

## 4. 타이틀 화면

타이틀 화면도 단순 로고와 버튼에서 `작전 단말기`로 변경합니다.

- HERO STRIKE 로고
- 현재 작전 카드
- 시스템 요약 칩: MISSIONS, BUILD TREE, BOSS CORE, ARMORY
- 계정 Blueprint 진행
- `ENTER COMMAND DECK`

## 5. 구현 원칙

- Canvas 420×760 유지
- 기존 트레이서 스프라이트 재사용
- 로비 선택 탭은 WeakMap 런타임으로 관리
- 게임 전투 상태 타입에는 UI 전용 필드를 추가하지 않음
- 레이아웃, 데이터 모델, 카드 렌더러, 로비 렌더러를 분리
- 잠긴 Blueprint 선택 차단 규칙은 기존 로직 유지
- 모바일 터치 영역은 최소 44px 이상 확보
