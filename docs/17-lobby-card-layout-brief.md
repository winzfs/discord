# 17. 로비 영웅 카드 레이아웃 정리 브리핑

## 1. 목적

모바일 로비의 영웅 카드 UI가 깨지는 문제를 정리했습니다.

추가로 과하게 늘어진 카드 비율과 들쭉날쭉한 내부 여백을 줄여, 화려함보다는 실제 모바일 게임 카드처럼 정돈된 형태로 다듬었습니다.

주요 문제:

- 신화 카드 glow 수정 이후 스프라이트 시트 일부가 카드 밖으로 노출됨
- 영웅 이름이 카드 폭을 넘거나 하단 정보와 겹침
- 레벨/역할 텍스트와 조각 수량 배지가 겹침
- 하단 바텀 내비게이션이 카드 마지막 줄을 가림
- 카드 높이가 길고 내부 기준선이 흔들려 게임 UI처럼 보이지 않음

## 2. 적용 파일

```text
apps/web/src/styles/lobby-mythic-glow.css
apps/web/src/styles/lobby-card-layout-fix.css
apps/web/src/styles/lobby-stage-polish.css
```

## 3. 변경 내용

### 3.1 신화 카드 스프라이트 클리핑 복구

`hero-portrait`에 들어갔던 `overflow: visible`을 다시 `overflow: hidden`으로 되돌렸습니다.

신화 glow는 캐릭터 이미지가 새어 나오지 않는 `box-shadow` 중심으로 유지합니다.

### 3.2 카드 내부 세로 레이아웃 고정

영웅/유물 카드를 grid 레이아웃으로 정리했습니다.

순서:

```text
등급 배지
초상화
이름
레벨/역할 또는 설명
조각 수량
```

카드 높이는 기존보다 줄이고, 각 영역의 행 높이를 고정해 카드마다 기준선이 흔들리지 않게 했습니다.

### 3.3 텍스트 오버플로우 방지

- 이름은 최대 2줄 표시
- 너무 긴 이름은 줄바꿈/클램프 처리
- 레벨/역할은 1줄 말줄임 처리
- 조각 수량은 일반 흐름 안의 하단 배지로 배치
- 글자 그림자는 줄여서 모바일에서 덜 지저분하게 보이도록 조정

### 3.4 카드 하단 정보 패널 느낌 추가

카드 하단에 아주 약한 내부 패널 레이어를 추가했습니다.

목적은 장식 추가가 아니라, 이름/레벨/조각 정보가 하나의 정보 영역처럼 읽히게 만드는 것입니다.

### 3.5 바텀 내비게이션 가림 방지

`lobby-shell`의 하단 padding을 늘려 마지막 카드가 바텀 내비게이션에 가려지지 않게 했습니다.

## 4. 확인 필요

```bash
pnpm typecheck
pnpm build:web
```

브라우저 확인 항목:

- 신화 카드에서 캐릭터가 카드 밖으로 반복 노출되지 않는지
- 일반/희귀/영웅/전설/신화 카드에서 이름이 카드 밖으로 나가지 않는지
- 카드별 이름/레벨/조각 수량의 기준선이 일정한지
- 레벨/역할 텍스트와 조각 배지가 겹치지 않는지
- 미보유 카드의 `비활성` 배지가 카드 안에서 안정적으로 보이는지
- 마지막 줄 카드가 하단 내비게이션에 가려지지 않는지
