# 03. 게임 시스템 설계

## 1. 시스템 개요

이 문서는 랜덤 협동 디펜스 웹게임의 내부 게임 시스템을 정리합니다.

핵심 시스템:

- 유닛 시스템
- 랜덤 호출 시스템
- 합성/승급 시스템
- 강화 시스템
- 웨이브/적 시스템
- 보스 시스템
- 점수/보상 시스템
- 유물 시스템
- 마스코트 시스템
- 미션/시즌 시스템

MVP에서는 모든 시스템을 완성하지 않고, 핵심 플레이가 돌아가는 최소 범위부터 구현합니다.

## 2. 유닛 시스템

### 2.1 유닛 데이터 구조

유닛은 코드에 직접 하드코딩하지 않고 데이터 파일 또는 DB에서 관리합니다.

예상 필드:

```ts
type UnitDefinition = {
  id: string;
  name: string;
  role: 'attack' | 'support' | 'control' | 'luck' | 'defense';
  grade: 'common' | 'rare' | 'epic' | 'legendary' | 'mythic';
  attackType: 'single' | 'area' | 'chain' | 'dot' | 'buff' | 'debuff';
  baseDamage: number;
  attackSpeed: number;
  range: number;
  skillId?: string;
  tags: string[];
};
```

실제 구현 시에는 `src/game/data/units.ts` 또는 `src/game/data/units.json`으로 분리합니다.

### 2.2 유닛 역할

#### 공격형

- 기본 피해를 담당합니다.
- 단일 대상, 범위 공격, 연쇄 공격 등으로 구분합니다.

예시:

- 펄스 슈터: 빠른 단일 공격
- 폭발 담당: 느리지만 범위 피해
- 레이저 드론: 지속 피해

#### 지원형

- 아군의 공격 속도, 공격력, 자원 획득량을 올립니다.
- 직접 공격력은 낮게 설정합니다.

예시:

- 부스터: 주변 유닛 공격 속도 증가
- 에너지 매니저: 처치 시 자원 추가 획득
- 응원봇: 일정 시간마다 전체 버프

#### 제어형

- 적을 느리게 하거나 멈추게 합니다.
- 보스에게는 효과가 감소하도록 설계합니다.

예시:

- 슬로우 해커: 적 이동속도 감소
- 스턴 비콘: 확률적으로 기절
- 방해 전파기: 방어력 감소

#### 행운형

- 확률 효과로 대박을 노리는 역할입니다.
- 게임의 운빨 감성을 강화합니다.

예시:

- 럭키 호출러: 낮은 확률로 호출 비용 환급
- 도박꾼 봇: 낮은 확률로 높은 피해
- 복권 드론: 합성 시 추가 보상 확률

MVP에서는 행운형을 1~2개만 넣는 것을 추천합니다.

## 3. 유닛 등급과 성장

### 3.1 등급 구조

추천 등급:

| 등급 | 내부값 | 역할 |
|---|---|---|
| 일반 | common | 자주 등장하는 기본 유닛 |
| 특수 | rare | 초반 전투력 담당 |
| 에이스 | epic | 빌드 핵심 |
| 전설 | legendary | 강력한 판도 변화 |
| 비밀병기 | mythic | 추후 이벤트/고급 콘텐츠용 |

MVP에서는 `common`, `rare`, `epic`, `legendary`만 구현합니다.

### 3.2 등급별 등장 확률 예시

초기 호출 확률 예시입니다. 실제 수치는 밸런스 파일에서 조정합니다.

| 등급 | 확률 |
|---|---:|
| 일반 | 65% |
| 특수 | 25% |
| 에이스 | 8% |
| 전설 | 2% |

게임 진행, 유물, 이벤트 규칙에 따라 확률이 변경될 수 있습니다.

## 4. 랜덤 호출 시스템

### 4.1 기본 규칙

- 호출 버튼을 누르면 자원을 소모합니다.
- 빈 슬롯이 있을 때만 호출할 수 있습니다.
- 호출 결과는 등급 확률표와 유닛 풀에 따라 결정됩니다.
- 호출 비용은 호출 횟수에 따라 조금씩 증가할 수 있습니다.

### 4.2 호출 비용 증가

예시:

```text
현재 호출 비용 = 기본 비용 + 호출 횟수 * 증가량
```

MVP 예시:

- 기본 비용: 10
- 증가량: 2
- 최대 비용: 100

### 4.3 호출 결과 결정 순서

1. 등급 확률표에서 등급 결정
2. 해당 등급의 유닛 목록 필터링
3. 유닛 하나 랜덤 선택
4. 빈 슬롯에 배치
5. 등장 연출 표시

### 4.4 천장 시스템 여부

초기 MVP에서는 복잡한 천장 시스템을 넣지 않습니다.

추후 추가 가능:

- 일정 횟수 이상 전설 미등장 시 확률 증가
- 이벤트 기간 특정 역할 확률 증가
- 서버 버프에 따라 특정 등급 보정

## 5. 합성/승급 시스템

### 5.1 기본 합성 규칙

추천 규칙:

- 같은 등급 유닛 3개를 합성하면 상위 등급 유닛 1개 생성
- 합성 결과 유닛은 같은 역할 또는 완전 랜덤 중 하나로 설계 가능

MVP 추천:

```text
같은 등급 3개 -> 상위 등급 랜덤 유닛 1개
```

이유:

- 구현이 쉽다.
- 직관적이다.
- 운빨 요소가 강하다.
- 필드 정리 역할을 한다.

### 5.2 합성 결과 방식

#### 완전 랜덤형

- 상위 등급 전체 풀에서 랜덤
- 운빨 재미가 큼
- 원하는 빌드 만들기 어려움

#### 역할 유지형

- 같은 역할 유닛끼리 합성하면 같은 역할 상위 유닛 등장
- 전략성이 큼
- 구현과 UI가 조금 복잡함

#### 절충형

- 같은 역할이 나올 확률 70%
- 다른 역할이 나올 확률 30%

MVP에서는 완전 랜덤형으로 시작하고, 이후 역할 유지형으로 확장 가능합니다.

### 5.3 합성 UI

- 합성 가능한 유닛은 테두리 표시
- 같은 등급 3개 이상 보유 시 합성 버튼 활성화
- 자동 합성 버튼은 추후 추가
- 합성 결과는 짧은 팝업 또는 슬롯 이펙트로 표시

## 6. 강화 시스템

### 6.1 전투 중 강화

전투 중 자원을 사용해 임시 강화합니다.

예시:

- 전체 공격력 +5%
- 전체 공격속도 +3%
- 특정 역할 공격력 증가
- 호출 등급 확률 소폭 증가

MVP 추천:

- 전체 공격력 강화 1종만 구현
- 비용은 단계별 증가

### 6.2 계정 성장 강화

게임 밖에서 누적되는 장기 성장입니다.

예시:

- 시작 자원 증가
- 기본 공격력 증가
- 특정 역할 등장 확률 증가
- 일일 보상 증가

MVP에서는 적용하지 않거나, 유물 시스템 문서만 준비합니다.

## 7. 적/웨이브 시스템

### 7.1 적 데이터 구조

```ts
type EnemyDefinition = {
  id: string;
  name: string;
  type: 'normal' | 'fast' | 'tank' | 'split' | 'boss';
  hp: number;
  speed: number;
  reward: number;
  damageToLife: number;
  skillId?: string;
};
```

### 7.2 웨이브 데이터 구조

```ts
type WaveDefinition = {
  wave: number;
  enemies: Array<{
    enemyId: string;
    count: number;
    spawnInterval: number;
  }>;
  bossId?: string;
  rewardOnClear: number;
};
```

### 7.3 웨이브 설계 원칙

- 1~5웨이브: 튜토리얼/쉬운 구간
- 6~10웨이브: 합성 필요성 체감
- 11~20웨이브: 역할 조합 중요
- 21웨이브 이후: 랭킹 경쟁 구간
- 5 또는 10웨이브마다 보스 등장

MVP에서는 30웨이브까지 데이터를 만들고, 이후는 계수 증가 방식으로 반복할 수 있습니다.

## 8. 보스 시스템

### 8.1 보스 역할

보스는 단순히 체력 높은 적이 아니라, 플레이어에게 선택을 강요하는 장치입니다.

보스 패턴 예시:

- 빠르게 이동
- 일정 시간 방어력 증가
- 주변 적 이동속도 증가
- 유닛 공격속도 감소 디버프
- 처치 제한 시간 존재

MVP에서는 다음 2종만 추천합니다.

1. 대형 버그: 체력 높은 기본 보스
2. 스피드 크래셔: 이동속도 빠른 보스

## 9. 점수 시스템

### 9.1 점수 요소

- 도달 웨이브
- 처치 수
- 보스 처치 수
- 남은 생명
- 플레이 시간
- 최고 등급 유닛 수
- 이벤트 보너스

### 9.2 MVP 점수 공식

```text
score = wave * 100 + kills * 5 + bossKills * 500 + remainingLife * 100
```

이 공식은 단순하고 설명하기 쉽습니다.

추후 시즌마다 점수 공식을 바꿀 수 있도록 `scoreRules`로 분리합니다.

## 10. 보상 시스템

### 10.1 전투 보상

- 계정 경험치
- 시즌 포인트
- 유물 조각
- 마스코트 조각
- 칭호 조건 달성

MVP에서는 다음만 저장합니다.

- 플레이 기록
- 최고 점수
- 시즌 점수
- 총 플레이 횟수

### 10.2 보상 지급 원칙

- 신규 유저와 기존 유저의 격차가 너무 커지지 않게 한다.
- 커뮤니티 이벤트 보상이 게임 밸런스를 무너뜨리지 않게 한다.
- 외형/칭호 보상을 적극 활용한다.

## 11. 유물 시스템

### 11.1 유물의 역할

유물은 계정 단위 패시브 강화입니다.

예시:

| 유물 후보 | 효과 |
|---|---|
| 낡은 서버 칩 | 시작 자원 증가 |
| 응급 패치 노트 | 생명 +1 |
| 랜덤 부스터 | 호출 비용 1% 감소 |
| 버그 추적기 | 보스 피해량 증가 |
| 행운 로그 | 전설 등장 확률 소폭 증가 |

### 11.2 구현 단계

MVP:

- 유물 데이터 구조만 준비
- 실제 효과 미적용 가능

1차 확장:

- 유물 10개
- 레벨 1~5
- 보상으로 조각 획득

2차 확장:

- 시즌 한정 유물
- 이벤트 유물
- 유물 프리셋

## 12. 마스코트/펫 시스템

### 12.1 역할

마스코트는 플레이어 옆에서 보조 효과를 주는 수집 요소입니다.

예시:

| 마스코트 | 효과 |
|---|---|
| 미니봇 | 처치 자원 +1% |
| 잠꾸러기 고양이 | 낮은 확률로 합성 비용 환급 |
| 응원 햄스터 | 시작 생명 +1 |
| 오류 먹는 강아지 | 보스 처치 보상 증가 |

### 12.2 MVP 적용 여부

MVP에서는 제외합니다.

다만 DB와 폴더 구조에서 추후 추가 가능하도록 `companions` 또는 `mascots` 도메인을 예약합니다.

## 13. 미션 시스템

### 13.1 일일 미션

예시:

- 게임 1회 플레이
- 10웨이브 도달
- 보스 1회 처치
- 합성 5회 진행
- 전설 유닛 1회 획득

### 13.2 주간 미션

예시:

- 누적 점수 50,000점
- 게임 10회 플레이
- 시즌 랭킹 참여
- 특정 역할 유닛으로 보스 처치

MVP에서는 미션 없이 시작하고, 대시보드에 `준비 중` 영역만 둘 수 있습니다.

## 14. 시즌 시스템

### 14.1 시즌 목적

- 랭킹 초기화로 반복 참여 유도
- 이벤트 운영에 적합
- 보상 지급 기준 제공

### 14.2 시즌 데이터

```ts
type Season = {
  id: string;
  name: string;
  startsAt: string;
  endsAt: string;
  ruleSetId: string;
  status: 'scheduled' | 'active' | 'ended';
};
```

### 14.3 시즌 보상

- 랭킹 칭호
- 프로필 배지
- 마스코트 조각
- 유물 조각
- 디스코드 역할 지급 후보

MVP에서는 시즌 기간과 랭킹만 구현합니다.

## 15. 데이터 분리 원칙

게임 밸런스와 규칙은 코드 내부에 흩어지면 안 됩니다.

권장 분리:

```text
src/game/data/units.ts
src/game/data/enemies.ts
src/game/data/waves.ts
src/game/data/relics.ts
src/game/data/mascots.ts
src/game/data/balance.ts
src/game/rules/scoring.ts
src/game/rules/summon.ts
src/game/rules/merge.ts
```

이렇게 해야 Codex가 이후 기능을 추가할 때 특정 파일만 수정하면 됩니다.

## 16. MVP 시스템 범위 최종안

MVP에서 실제 구현할 시스템:

- 유닛 12개
- 등급 4개
- 적 3종
- 보스 2종
- 웨이브 30개
- 랜덤 호출
- 합성
- 전체 공격력 강화
- 점수 계산
- 결과 저장
- 랭킹 표시

MVP에서 문서/구조만 준비할 시스템:

- 유물
- 마스코트
- 미션
- 시즌 보상
- 듀오 협동
- 서버 레이드
