# 14. 같은 캐릭터 중첩 보드 및 이동 시스템 기획

## 1. 목적

업로드된 `운빨존많겜류 게임시스템 기획문서.docx`를 기준으로, 현재 게임을 단순한 1칸 1유닛 랜덤 디펜스가 아니라 `랜덤 소환 + 같은 캐릭터 3마리 중첩 + 드래그 이동 + 자리 교환 + 합성/판매 메뉴 + 자동 전투` 중심의 운영형 랜덤 디펜스로 확장하기 위한 세부 기획입니다.

이 문서는 원본 시스템을 그대로 복제하지 않고, 우리 디스코드 웹게임에 맞춘 독자 시스템 구조로 정리합니다.

## 2. 현재 구현 상태 요약

2026-06 기준으로 `/play`에 다음 기능이 구현되어 있습니다.

| 영역 | 상태 | 구현 내용 |
|---|---|---|
| 중첩 보드 | 구현 완료 | `BoardCell[]` 구조, 칸당 최대 3마리 |
| 중첩 조건 | 구현 완료 | 같은 캐릭터만 같은 칸에 중첩 가능 |
| 소환 배치 | 구현 완료 | 같은 캐릭터 칸 우선, 없으면 빈 칸 |
| 도박 배치 | 구현 완료 | 도박으로 나온 유닛도 같은 중첩 규칙 적용 |
| 전투력 계산 | 구현 완료 | 모든 칸의 모든 유닛 전투력 합산 |
| 그래픽 중첩 표현 | 구현 완료 | `x2`, `x3` 텍스트 없이 유닛 그래픽 개수로 표현 |
| 드래그 이동 | 구현 완료 | 유닛 칸을 드래그해서 목적지 칸에 놓음 |
| 빈 칸 이동 | 구현 완료 | 목적지가 빈 칸이면 유닛 1마리 이동 |
| 같은 캐릭터 중첩 | 구현 완료 | 목적지가 같은 캐릭터 칸이고 3마리 미만이면 중첩 |
| 다른 캐릭터 자리 교환 | 구현 완료 | 목적지가 다른 캐릭터 칸이면 두 칸의 스택 위치를 교환 |
| 이동 연출 | 구현 완료 | 드롭 후 유닛 고스트가 목적지까지 걸어가는 연출 |
| 유닛 메뉴 | 구현 완료 | 짧게 탭/클릭 시 유닛 위에 작은 메뉴 표시 |
| 합성 메뉴 | 구현 완료 | 같은 캐릭터 3중첩일 때 합성 버튼 활성화 |
| 판매 메뉴 | 구현 완료 | 선택 칸의 맨 위 유닛 1마리 판매 |
| 전투 연출 | 미구현 | 적 이동/유닛 공격/투사체 연출 필요 |
| 오버드라이브 조합 | 미구현 | 신화 버튼 실제 조합 팝업 필요 |

## 3. 핵심 규칙 확정

중첩은 같은 등급끼리 가능한 것이 아닙니다.

```text
같은 캐릭터만 같은 칸에 중첩 가능
한 칸 최대 3마리까지 가능
서로 다른 캐릭터는 같은 등급이어도 같은 칸에 들어갈 수 없음
```

예시:

| 상황 | 가능 여부 | 설명 |
|---|---|---|
| 캐서디 + 캐서디 | 가능 | 같은 캐릭터 |
| 캐서디 + 캐서디 + 캐서디 | 가능 | 같은 캐릭터 3중첩 |
| 캐서디 + 벤처 | 같은 칸 중첩 불가 | 드롭 시 두 칸 자리 교환 |
| 일반 딜러 + 일반 탱커 | 같은 칸 중첩 불가 | 같은 등급이어도 캐릭터가 다름 |
| 희귀 탱커 + 희귀 탱커 | 가능 | 같은 캐릭터라면 가능 |

## 4. 보드 데이터 구조

기존 구조는 한 칸에 하나의 유닛만 들어갈 수 있었습니다.

```ts
board: Array<BoardHero | null>
```

현재 구현된 목표 구조는 다음과 같습니다.

```ts
export const MAX_STACK_PER_CELL = 3;

type BoardCell = {
  position: {
    row: number;
    column: number;
  };
  heroId: string | null;
  units: BoardHero[];
};

type GameState = {
  board: BoardCell[];
};
```

`heroId`는 해당 칸이 어떤 캐릭터 전용으로 사용 중인지 나타냅니다.

```text
빈 칸: heroId = null, units = []
캐서디 1마리: heroId = "cassidy", units.length = 1
캐서디 3마리: heroId = "cassidy", units.length = 3
```

구현 파일:

```text
packages/game/src/types/hero.ts
packages/game/src/types/gameState.ts
packages/game/src/systems/boardSystem.ts
```

## 5. 소환/도박 배치 규칙

소환 또는 도박으로 유닛이 생성되면 다음 순서로 배치 위치를 찾습니다.

```text
1. 새로 뽑힌 유닛과 같은 캐릭터가 있고 3마리 미만인 칸 우선
2. 그런 칸이 없으면 빈 칸 사용
3. 빈 칸도 없으면 소환 실패
```

이 규칙의 장점:

- 같은 캐릭터가 자연스럽게 한 칸에 모입니다.
- 보드가 덜 어지럽습니다.
- 3중첩 완성의 목표감이 생깁니다.
- 캐릭터별 보유 수를 직관적으로 이해할 수 있습니다.

주의:

- 다른 캐릭터가 있는 칸에는 자동 배치하지 않습니다.
- 같은 등급이어도 캐릭터가 다르면 같은 칸에 자동 중첩하지 않습니다.
- 같은 캐릭터 3마리 칸이 이미 꽉 찼다면, 같은 캐릭터가 새로 나와도 빈 칸을 사용합니다.

구현 파일:

```text
packages/game/src/systems/summonSystem.ts
packages/game/src/systems/gambleSystem.ts
```

## 6. 드래그 이동/중첩/자리 교환 규칙

유닛 이동은 소환 운을 운영으로 보정하는 핵심 시스템입니다.

현재 `/play`는 드래그 앤 드롭을 기본 조작으로 사용합니다.

```text
유닛이 있는 칸을 드래그
→ 목적지 칸에 드롭
→ 목적지 상태에 따라 이동/중첩/자리 교환
```

목적지별 처리:

| 목적지 상태 | 결과 |
|---|---|
| 빈 칸 | 선택 칸의 맨 위 유닛 1마리 이동 |
| 같은 캐릭터, 3마리 미만 | 선택 칸의 맨 위 유닛 1마리 중첩 |
| 같은 캐릭터, 이미 3마리 | 이동 실패, `칸이 가득 찼어` 표시 |
| 다른 캐릭터 | 두 칸의 스택 전체 위치 교환 |
| 보드 밖 | 이동 취소 |

자리 교환 규칙:

```text
다른 캐릭터 칸에 드롭하면 유닛 1마리만 바꾸는 것이 아니라, 두 칸의 스택 전체가 서로 위치를 바꿉니다.
```

예시:

```text
A칸: 캐서디 x2
B칸: 벤처 x1

A칸을 B칸에 드롭
→ A칸: 벤처 x1
→ B칸: 캐서디 x2
```

이 방식은 서로 다른 캐릭터가 같은 칸에 섞이지 않게 하면서도, 플레이어가 원하는 배치를 빠르게 정리할 수 있게 해줍니다.

구현 파일:

```text
packages/game/src/systems/boardSystem.ts
apps/web/src/game-client/pixi/createPixiGame.ts
```

## 7. 이동 연출 규칙

이동은 즉시 순간이동처럼 보이지 않게 처리합니다.

현재 구현:

```text
드래그 중: 유닛 고스트가 손가락/마우스를 따라감
드롭 후: 유닛 고스트가 출발 칸에서 목적지 칸까지 걸어감
자리 교환: 양쪽 대표 유닛 고스트가 서로 반대 방향으로 걸어감
연출 종료: 실제 board state 반영
```

연출 세부:

- 이동 중 유닛이 위아래로 살짝 흔들립니다.
- 이동 중 좌우 회전이 약하게 들어갑니다.
- 이동 중에는 소환/도박/강화/웨이브 버튼과 보드 조작을 잠급니다.
- 연출 완료 후 이동/중첩/자리 교환 피드백 텍스트를 표시합니다.

## 8. 그래픽 중첩 표시 방식

중첩 상태는 `x2`, `x3` 같은 텍스트로 표시하지 않습니다.

텍스트 대신 칸 안의 유닛 그래픽 개수와 배치로 표현합니다.

```text
1중첩: 칸 중앙에 유닛 그래픽 1개
2중첩: 칸 안에 같은 유닛 그래픽 2개를 좌우 또는 대각선으로 배치
3중첩: 칸 안에 같은 유닛 그래픽 3개를 삼각형 형태로 배치
```

추천 배치:

```text
1중첩
   ●

2중첩
 ●   ●

3중첩
   ●
 ●   ●
```

시각 규칙:

- 숫자 텍스트는 사용하지 않습니다.
- 2중첩부터 유닛 그래픽 크기를 약간 줄여 칸 안에 모두 보이게 합니다.
- 3중첩 칸은 테두리를 더 밝게 하거나 미세한 반짝임을 줄 수 있습니다.
- 합성 가능한 3중첩 칸은 노란색 계열 테두리/글로우로 강조합니다.
- 등급 색상 테두리는 유지하되, 중첩 수는 그래픽 개수로만 표현합니다.

## 9. 유닛 메뉴: 합성/판매

짧게 탭/클릭하면 유닛 위에 작은 메뉴가 표시됩니다.

```text
유닛 칸 짧게 탭
→ [합성] [판매] 메뉴 표시
```

드래그와 메뉴는 다음 기준으로 구분합니다.

```text
짧게 탭/클릭: 메뉴 표시
8px 이상 이동: 드래그 이동으로 전환
```

### 9.1 합성

합성은 자동으로 발생하지 않습니다.

```text
같은 캐릭터 3중첩 칸 선택
→ 메뉴 표시
→ 합성 버튼 활성화
→ 합성 클릭
→ 3마리 소모
→ 상위 등급 랜덤 유닛 1마리 생성
```

합성 조건:

| 조건 | 설명 |
|---|---|
| 같은 캐릭터 | 3마리 모두 같은 `heroId`여야 함 |
| 같은 등급 | 3마리 모두 같은 `grade`여야 함 |
| 3중첩 | `units.length >= 3` |
| 최고 등급 아님 | 전설 등급은 현재 3중첩 합성 불가 |

현재 합성 결과는 `같은 캐릭터의 상위 등급`이 아니라 `상위 등급 풀에서 랜덤 1마리`입니다. 이 부분은 이후 밸런스에 따라 “같은 캐릭터 계열 승급” 구조로 바꿀 수 있습니다.

구현 파일:

```text
packages/game/src/systems/stackMergeSystem.ts
apps/web/src/game-client/pixi/createPixiGame.ts
```

### 9.2 판매

판매는 메뉴에서 항상 사용할 수 있습니다.

```text
유닛 칸 선택
→ 판매 클릭
→ 선택 칸의 맨 위 유닛 1마리 판매
→ 등급별 코인 지급
```

현재 판매 보상:

| 등급 | 판매 보상 |
|---|---:|
| 일반 | 8 코인 |
| 희귀 | 18 코인 |
| 영웅 | 45 코인 |
| 전설 | 110 코인 |

구현 파일:

```text
packages/game/src/systems/sellSystem.ts
apps/web/src/game-client/pixi/createPixiGame.ts
```

## 10. 전투력 계산

전투력 계산은 모든 칸의 모든 유닛을 대상으로 합니다.

```ts
const allUnits = board.flatMap((cell) => cell.units);
```

같은 캐릭터 3중첩이더라도 1마리로 계산하지 않습니다.

```text
캐서디 1마리 = 캐서디 전투력 1개분
캐서디 3마리 = 캐서디 전투력 3개분
```

현재 구현은 단순 합산입니다.

향후 추가 가능 요소:

```text
2중첩: 추가 보너스 없음
3중첩: 해당 칸 캐릭터 공격력 +10%
특정 유물/코어칩 보유 시 3중첩 보너스 증가
특정 오버드라이브 유닛이 3중첩 칸 강화
```

구현 파일:

```text
packages/game/src/systems/combatSystem.ts
```

## 11. 보드 크기

현재 `/play`는 5x4, 총 20칸 구조입니다.

같은 캐릭터 3중첩이 들어가면서 이론상 최대 유닛 수는 다음과 같습니다.

```text
20칸 x 3마리 = 최대 60마리
```

현재는 20칸 구조를 유지합니다.

추후 모바일 조작이 복잡하다고 판단되면 다음 대안을 검토합니다.

| 안 | 보드 | 최대 유닛 | 특징 |
|---|---:|---:|---|
| A | 3x5 | 45 | 모바일에 적합, 조작 단순 |
| B | 4x5 | 60 | 현재 구조 유지, 운영 깊이 높음 |

## 12. 현재 플레이 루프

현재 `/play` 기준 루프는 다음과 같습니다.

```text
게임 시작
→ 코인/행운석 지급
→ 소환 버튼으로 랜덤 유닛 획득
→ 같은 캐릭터 칸 우선 중첩
→ 유닛을 드래그해 이동/중첩/자리 교환
→ 짧게 탭해 합성/판매 메뉴 사용
→ 3중첩 칸은 합성으로 상위 등급 획득 가능
→ 판매로 공간과 코인 관리
→ 도박으로 고등급 유닛 노림
→ 공격력 강화로 전투력 상승
→ 웨이브 버튼으로 전투력 판정
```

## 13. 남은 주요 개발 과제

### 13.1 전투 연출

현재 웨이브는 전투력 기반 판정을 사용하며, 실제 전투 연출은 부족합니다.

다음 구현 목표:

```text
웨이브 버튼 클릭
→ 적들이 길을 따라 이동
→ 보드 유닛이 자동 공격 연출
→ 투사체/타격 이펙트 표시
→ 기존 전투력 판정 결과에 따라 웨이브 성공/누수/패배 처리
```

### 13.2 신화/오버드라이브 조합

현재 신화 버튼은 안내 메시지만 표시합니다.

다음 구현 목표:

```text
신화 버튼 클릭
→ 오버드라이브 조합 메뉴 표시
→ 제작 가능한 조합 밝게 표시
→ 재료가 부족한 조합은 비활성화
→ 제작 시 보드 위 재료 유닛 소모
→ 오버드라이브 유닛 생성
```

### 13.3 합성 결과 개선

현재 3중첩 합성 결과는 상위 등급 랜덤 유닛입니다.

추후 개선 후보:

```text
같은 캐릭터 계열 승급
역할 유지 랜덤 승급
등급별 합성 실패/대성공
3중첩 유지 보너스와 합성 선택의 전략성 강화
```

### 13.4 메뉴 UX 개선

현재 메뉴는 `[합성] [판매]` 기본 형태입니다.

추후 개선 후보:

```text
캐릭터 이름/등급 표시
판매 보상 미리보기
합성 결과 등급 미리보기
3중첩 칸에 합성 가능 아이콘 표시
메뉴 외부 터치 시 닫기
```

## 14. 밸런스 원칙

- 같은 캐릭터가 3중첩되면 기분이 좋아야 합니다.
- 3중첩이 항상 자동 합성되지 않아야 합니다.
- 3중첩을 전투용으로 유지할지, 합성 재료로 쓸지 선택하게 해야 합니다.
- 다른 캐릭터와 자리 교환을 통해 배치 정리를 빠르게 할 수 있어야 합니다.
- 필드가 꽉 차도 판매와 자리 교환으로 해결할 여지가 있어야 합니다.
- 초반에는 유닛 수 확보, 중반에는 같은 캐릭터 정리, 후반에는 핵심 캐릭터 3중첩과 오버드라이브 조합이 중요해야 합니다.
- 이동/판매/합성 판단으로 운이 나쁜 판도 어느 정도 복구 가능해야 합니다.
