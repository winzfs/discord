# 14. 같은 캐릭터 중첩 보드 및 이동 시스템 기획

## 1. 목적

업로드된 `운빨존많겜류 게임시스템 기획문서.docx`를 기준으로, 현재 게임을 단순한 1칸 1유닛 랜덤 디펜스가 아니라 `랜덤 소환 + 같은 캐릭터 3마리 중첩 + 유닛 이동 + 자동 전투 + 웨이브 방어` 중심의 운영형 랜덤 디펜스로 확장하기 위한 세부 기획입니다.

이 문서는 원본 시스템을 그대로 복제하지 않고, 우리 디스코드 웹게임에 맞춘 독자 시스템 구조로 정리합니다.

## 2. 핵심 규칙 확정

중첩은 같은 등급끼리 가능한 것이 아닙니다.

중첩 가능 조건은 다음과 같습니다.

```text
같은 캐릭터만 같은 칸에 중첩 가능
한 칸 최대 3마리까지 가능
서로 다른 캐릭터는 같은 등급이어도 같은 칸에 들어갈 수 없음
```

예시:

| 상황 | 가능 여부 | 설명 |
|---|---|---|
| 캐서디 + 캐서디 | 가능 | 같은 캐릭터 |
| 캐서디 + 캐서디 + 캐서디 | 가능 | 같은 캐릭터 3중첩 |
| 캐서디 + 벤처 | 불가 | 서로 다른 캐릭터 |
| 일반 딜러 + 일반 탱커 | 불가 | 같은 등급이어도 캐릭터가 다름 |
| 희귀 탱커 + 희귀 탱커 | 가능 | 같은 캐릭터라면 가능 |

## 3. 기존 구조와 목표 구조

### 3.1 기존 구조

현재 저장소의 보드는 한 칸에 하나의 유닛만 들어갈 수 있는 구조입니다.

```ts
board: Array<BoardHero | null>
```

문제점:

- 한 칸에 하나만 들어가서 중첩 운영이 불가능합니다.
- 같은 캐릭터를 모으는 재미가 약합니다.
- 유닛 이동/정리의 의미가 작습니다.
- 조합 재료를 보관하는 플레이가 어렵습니다.

### 3.2 목표 구조

```ts
export const MAX_STACK_PER_CELL = 3;

type BoardCell = {
  position: {
    row: number;
    column: number;
  };
  heroId: string | null;
  units: BoardHero[];
};

type GameState = {
  board: BoardCell[];
};
```

`heroId`는 해당 칸이 어떤 캐릭터 전용으로 사용 중인지 나타냅니다.

- 빈 칸: `heroId = null`, `units = []`
- 캐서디 1마리: `heroId = "cassidy"`, `units.length = 1`
- 캐서디 3마리: `heroId = "cassidy"`, `units.length = 3`

## 4. 소환 배치 규칙

소환 버튼을 누르면 다음 순서로 배치 위치를 찾습니다.

```text
1. 새로 뽑힌 유닛과 같은 캐릭터가 있고 3마리 미만인 칸 우선
2. 그런 칸이 없으면 빈 칸 사용
3. 빈 칸도 없으면 소환 실패
```

이 규칙의 장점:

- 같은 캐릭터가 자연스럽게 한 칸에 모입니다.
- 보드가 덜 어지럽습니다.
- 3중첩 완성의 목표감이 생깁니다.
- 캐릭터별 보유 수를 직관적으로 이해할 수 있습니다.

주의:

- 다른 캐릭터가 있는 칸에는 절대 배치하지 않습니다.
- 같은 등급이라도 캐릭터가 다르면 배치하지 않습니다.

## 5. 유닛 이동 규칙

유닛 이동은 소환 운을 운영으로 보정하는 핵심 시스템입니다.

기본 흐름:

```text
유닛 선택
→ 대상 칸 선택 또는 드래그
→ 대상 칸 상태 확인
→ 빈 칸이면 이동 가능
→ 같은 캐릭터가 있고 3마리 미만이면 이동 가능
→ 다른 캐릭터가 있으면 이동 불가
→ 3마리가 이미 있으면 이동 불가
```

이동 결과:

- 이동 후 원래 칸에 유닛이 0마리 남으면 해당 칸의 `heroId`는 `null`로 초기화합니다.
- 이동 후 대상 칸이 빈 칸이었다면 대상 칸의 `heroId`는 이동한 유닛의 `heroId`로 설정합니다.

## 6. 중첩이 만드는 플레이

같은 캐릭터 3중첩은 단순 보관이 아니라 플레이 목표입니다.

플레이어는 다음 판단을 하게 됩니다.

- 이 캐릭터를 계속 모을지
- 2중첩 상태를 유지하고 재료로 쓸지
- 좋은 위치로 옮겨 3중첩 화력 구역을 만들지
- 필요 없는 캐릭터를 판매/합성해서 공간을 만들지
- 보스 전에 특정 딜러 3중첩을 완성할지

## 7. 합성/조합과의 관계

같은 캐릭터 중첩은 합성과 직접 연결할 수 있습니다.

### 7.1 같은 캐릭터 3중첩 합성

```text
같은 캐릭터 3마리 중첩
→ 상위 등급 또는 강화된 형태로 변환
```

MVP에서는 자동 합성하지 않고, 플레이어가 버튼 또는 팝업에서 선택하도록 합니다.

이유:

- 3중첩 상태 자체가 전투에 도움이 될 수 있기 때문입니다.
- 플레이어가 전투용 3중첩으로 쓸지, 합성 재료로 쓸지 선택하게 해야 합니다.

### 7.2 오버드라이브 조합

우리 게임의 독자 고급 제작 시스템입니다.

```text
특정 캐릭터 또는 특정 역할/등급 재료
+ 행운석
→ 오버드라이브 유닛 제작
```

조합 재료는 보드 위에 존재해야 하며, 중첩/이동 시스템으로 재료를 정리하는 플레이를 유도합니다.

## 8. 보드 크기

현재 배포된 `/play`는 5x4, 총 20칸 구조입니다.

같은 캐릭터 3중첩이 들어가면 이론상 최대 유닛 수는 다음과 같습니다.

```text
20칸 x 3마리 = 최대 60마리
```

초기 모바일 MVP 기준으로는 복잡할 수 있으므로, 다음 두 가지 안을 비교합니다.

| 안 | 보드 | 최대 유닛 | 특징 |
|---|---:|---:|---|
| A | 3x5 | 45 | 모바일에 적합, 조작 단순 |
| B | 4x5 | 60 | 현재 구조 유지, 운영 깊이 높음 |

현재는 B안을 유지하되, UI가 복잡하면 A안으로 줄입니다.

## 9. UI 표시 방식

한 칸에 같은 캐릭터가 여러 마리 있을 때는 다음처럼 보여줍니다.

```text
1중첩: 캐릭터 아이콘 1개
2중첩: 캐릭터 아이콘 + x2 또는 점 2개
3중첩: 캐릭터 아이콘 + x3 또는 점 3개
```

추천:

- 모바일에서는 `x2`, `x3` 표시가 가장 읽기 쉽습니다.
- PC에서는 작은 유닛 마커 2~3개를 칸 안에 배치할 수 있습니다.
- 3중첩 칸은 테두리를 더 밝게 하거나 반짝임을 줄 수 있습니다.

## 10. 선택/이동 UI

### 10.1 모바일 우선

```text
1. 칸 터치
2. 해당 칸의 유닛 선택
3. 이동할 칸 터치
4. 이동 가능하면 이동
5. 불가능하면 메시지 표시
```

칸에 2~3마리가 있어도 모두 같은 캐릭터이므로, MVP에서는 `칸 단위 이동` 또는 `맨 위 유닛 1마리 이동`으로 단순화할 수 있습니다.

권장 MVP:

```text
칸 선택 → 대상 칸 선택 → 선택 칸에서 유닛 1마리 이동
```

### 10.2 PC 확장

이후 드래그 앤 드롭을 추가합니다.

```text
유닛 또는 칸 드래그
→ 대상 칸 위에 드롭
→ 같은 캐릭터/빈 칸/최대 중첩 여부 판정
```

## 11. 전투력 계산

전투력 계산은 모든 칸의 모든 유닛을 대상으로 합니다.

```ts
const allUnits = board.flatMap((cell) => cell.units);
```

같은 캐릭터 3중첩이더라도 1마리로 계산하지 않습니다.

```text
캐서디 1마리 = 캐서디 전투력 1개분
캐서디 3마리 = 캐서디 전투력 3개분
```

이후 3중첩 보너스를 추가할 수 있습니다.

예시:

```text
2중첩: 추가 보너스 없음
3중첩: 해당 칸 캐릭터 공격력 +10%
```

단, MVP에서는 보너스 없이 단순 합산으로 시작합니다.

## 12. MVP 우선순위

이 문서 기준의 새로운 MVP 우선순위는 다음과 같습니다.

1. `BoardCell` 타입 추가
2. `GameState.board`를 `BoardCell[]`로 변경
3. 소환 시 같은 캐릭터 칸 우선 배치
4. 빈 칸 배치
5. 모든 칸이 다른 캐릭터로 꽉 차 있거나 3중첩이면 소환 실패
6. 전투력 계산이 모든 중첩 유닛을 반영
7. 보드 UI에 `x1/x2/x3` 표시
8. 칸 선택 → 대상 칸 선택 이동
9. 이동 불가 메시지
10. 같은 캐릭터 3중첩 기반 합성
11. 오버드라이브 조합
12. 협동 모드

## 14. 중요한 밸런스 원칙

- 같은 캐릭터가 3중첩되면 기분이 좋아야 합니다.
- 하지만 3중첩만 정답이면 안 됩니다.
- 3중첩을 전투용으로 유지할지, 합성 재료로 쓸지 선택하게 해야 합니다.
- 필드가 꽉 차도 이동과 정리로 해결할 여지가 있어야 합니다.
- 초반에는 유닛 수 확보, 중반에는 같은 캐릭터 정리, 후반에는 핵심 캐릭터 3중첩과 오버드라이브 조합이 중요해야 합니다.
